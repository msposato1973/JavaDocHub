SOLID Principles

The five fundamental principles of object-oriented programming

S - Single Responsibility

"A class should have only one reason to change"
❌ WRONG EXAMPLE
class User {
    void saveToDatabase() { }
    void sendEmail() { }
    void logOperation() { }
}
✅ CORRECT EXAMPLE
class User { }
class UserRepository { }
class EmailService { }
class LoggerService { }

O - Open/Closed

"Open for extension, closed for modification"
interface Shape { double area(); }
class Rectangle implements Shape { ... }
class Circle implements Shape { ... }
// New shapes? Just implement the interface!

L - Liskov Substitution

"Subclasses must be substitutable for their base classes"
// A subclass must not alter expected behavior
Rectangle r = new Square(); // WARNING: might violate LSP!

I - Interface Segregation

"Specific interfaces are better than a generic one"
interface Workable { void work(); }
interface Eatable { void eat(); }
// Robot implements only Workable, Human implements both

D - Dependency Inversion

"Depend on abstractions, not on concrete implementations"
interface MessageService { void send(String msg); }
class Notifier {
    private final MessageService service;
    Notifier(MessageService s) { this.service = s; }
}

Practical Example: Payment System

// All SOLID principles applied together
interface PaymentMethod { void pay(double amount); }
class CreditCard implements PaymentMethod { ... }
class PayPal implements PaymentMethod { ... }
// Dependency Injection via constructor
