Switch Expression

Java 11 → Only switch statement
Java 17 → Switch expression + yield
Java 11
String day = "MONDAY";
String type;
switch (day) {
    case "SATURDAY":
    case "SUNDAY":
        type = "Weekend";
        break;
    case "MONDAY":
    case "TUESDAY":
    case "WEDNESDAY":
    case "THURSDAY":
    case "FRIDAY":
        type = "Weekday";
        break;
    default:
        type = "Unknown";
}
System.out.println(type);
Java 17 NEW
String day = "MONDAY";
String type = switch (day) {
    case "SATURDAY", "SUNDAY" -> "Weekend";
    case "MONDAY", "TUESDAY", 
         "WEDNESDAY", "THURSDAY", 
         "FRIDAY" -> "Weekday";
    default -> "Unknown";
};
System.out.println(type);

// With yield for code blocks
String result = switch (day) {
    case "MONDAY" -> {
        System.out.println("Start of week");
        yield "Monday";
    }
    default -> "Other";
};
Java 17 advantage: 5 lines vs 16 lines for the same result!

Record Class (Data Carrier)

Java 11 → Classes with boilerplate (~50 lines)
Java 17 → Record (1 line)
Java 11
public class Person {
    private final String name;
    private final int age;
    
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    public String getName() { return name; }
    public int getAge() { return age; }
    
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Person)) return false;
        Person that = (Person) o;
        return age == that.age && 
               Objects.equals(name, that.name);
    }
    
    @Override
    public int hashCode() {
        return Objects.hash(name, age);
    }
    
    @Override
    public String toString() {
        return "Person{name='" + name + "', age=" + age + "}";
    }
}
Java 17 NEW
public record Person(String name, int age) {}

// Usage
Person p = new Person("Mario", 30);
System.out.println(p.name());  // automatic getter
System.out.println(p);  // automatic toString()

// Record with compact constructor (validation)
public record User(String email, int age) {
    public User {
        if (age < 0) 
            throw new IllegalArgumentException("Negative age");
        if (!email.contains("@")) 
            throw new IllegalArgumentException("Invalid email");
    }
}

// Record with additional methods
public record Point(int x, int y) {
    public double distanceFromOrigin() {
        return Math.sqrt(x * x + y * y);
    }
}
Java 17 advantage: 1 line of code vs ~50 lines for an immutable DTO!

Pattern Matching for instanceof

Java 11 → Explicit cast
Java 17 → Direct pattern binding
Java 11
Object obj = "Hello World";
if (obj instanceof String) {
    String s = (String) obj;  // explicit cast!
    System.out.println(s.toUpperCase());
}

// With additional conditions
if (obj instanceof String) {
    String s = (String) obj;
    if (s.length() > 5) {
        System.out.println("Long: " + s);
    }
}
Java 17 NEW
Object obj = "Hello World";
if (obj instanceof String s) {
    System.out.println(s.toUpperCase());
}

// With additional conditions (&&)
if (obj instanceof String s && s.length() > 5) {
    System.out.println("Long: " + s);
}

// Pattern matching in switch (Java 17 preview, 21 stable)
Object value = "Test";
String result = switch (value) {
    case Integer i -> "Integer: " + i;
    case String s -> "String: " + s;
    case null -> "Null";
    default -> "Unknown";
};

Sealed Class

Java 11 → Open inheritance (any class can extend)
Java 17 → Precise control over permitted subclasses
// Java 17 - Sealed Class
public sealed class Shape 
    permits Circle, Rectangle, Triangle {
    public abstract double area();
}

// Subclasses must be final, sealed, or non-sealed
public final class Circle extends Shape {
    private final double radius;
    
    public Circle(double radius) { this.radius = radius; }
    
    @Override
    public double area() { 
        return Math.PI * radius * radius; 
    }
}

public sealed class Rectangle extends Shape permits Square {
    private final double width;
    private final double height;
    
    public Rectangle(double width, double height) {
        this.width = width;
        this.height = height;
    }
    
    @Override
    public double area() { 
        return width * height; 
    }
}

public final class Square extends Rectangle {
    public Square(double side) {
        super(side, side);
    }
}

public non-sealed class Triangle extends Shape {
    // non-sealed allows unlimited extensions
    private final double base;
    private final double height;
    
    public Triangle(double base, double height) {
        this.base = base;
        this.height = height;
    }
    
    @Override
    public double area() { 
        return (base * height) / 2; 
    }
}

// Attempting to extend Shape with a non-permitted class causes an error!
// public class Pentagon extends Shape {}  // COMPILATION ERROR!
Java 17 advantage: Explicit control over which classes can extend

var in Lambda Parameters

Java 11 → Implicit type without annotations
Java 17 → var with annotations in lambdas
import javax.validation.constraints.NotNull;
import java.util.*;

public class VarLambdaDemo {
    public static void main(String[] args) {
        List<String> names = Arrays.asList("Alice", "Bob", "Charlie");
        
        // ========== Java 11 ==========
        // Lambda with inferred type (without var)
        names.sort((a, b) -> a.compareTo(b));
        
        // Cannot use annotations in lambda parameters
        // names.sort((@NotNull a, @NotNull b) -> a.compareTo(b));  // ERROR!
        
        // ========== Java 17 ==========
        // var can be used in lambda parameters
        names.sort((var a, var b) -> a.compareTo(b));
        
        // With annotations (useful for validation) - only with var!
        names.sort((@NotNull var a, @NotNull var b) -> a.compareTo(b));
        
        // var allows specifying type when needed
        Function<String, Integer> length = (var s) -> s.length();
        
        // Useful with complex generic types
        BiFunction<List<String>, Integer, String> get = 
            (var list, var index) -> list.get(index);
    }
}

New String Methods

Java 11 → strip(), isBlank(), lines(), repeat()
Java 17 → Same methods (more mature)
String text = "  \t Hello World \n  ";

// strip() vs trim() - important difference!
String strip = text.strip();   // "Hello World" (removes Unicode spaces)
String trim = text.trim();      // "Hello World" (ASCII only)

// isBlank() - checks if string is empty or only spaces
"   ".isBlank();  // true
"".isBlank();      // true

// lines() - split by lines (more efficient than split("\n"))
String multiline = "line1\nline2\nline3";
multiline.lines().forEach(System.out::println);

// repeat() - repeats the string
"Java".repeat(3);  // "JavaJavaJava"

// formatted() - printf-style formatting (Java 15+)
String name = "Mario";
int age = 30;
String msg = "Name: %s, Age: %d".formatted(name, age);
System.out.println(msg);  // "Name: Mario, Age: 30"

// indent() - adds/removes indentation (Java 12+)
String indented = "text".indent(4);  // "    text\n"

// transform() - transforms the string (Java 12+)
String result = "hello".transform(s -> s.toUpperCase());
System.out.println(result);  // "HELLO"

HTTP Client

Java 11 → Modern HttpClient (introduced)
Java 17 → Enhanced HttpClient (WebSocket, HTTP/2)
import java.net.http.*;
import java.net.URI;
import java.time.Duration;
import java.util.concurrent.*;

public class HttpClientDemo {
    public static void main(String[] args) throws Exception {
        
        // Modern HttpClient (introduced in Java 11, enhanced in Java 17)
        HttpClient client = HttpClient.newBuilder()
            .version(HttpClient.Version.HTTP_2)
            .connectTimeout(Duration.ofSeconds(10))
            .followRedirects(HttpClient.Redirect.NORMAL)
            .build();
        
        // Synchronous GET request
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create("https://api.example.com/users/1"))
            .timeout(Duration.ofSeconds(30))
            .header("Content-Type", "application/json")
            .GET()
            .build();
        
        HttpResponse<String> response = client.send(request, 
            HttpResponse.BodyHandlers.ofString());
        System.out.println("Status: " + response.statusCode());
        System.out.println("Body: " + response.body());
        
        // Asynchronous request (non-blocking)
        CompletableFuture<HttpResponse<String>> future = client.sendAsync(
            request, 
            HttpResponse.BodyHandlers.ofString()
        );
        future.thenAccept(resp -> System.out.println(resp.body()));
        
        // POST request with JSON
        String json = "{\"name\":\"Mario\",\"age\":30}";
        HttpRequest postRequest = HttpRequest.newBuilder()
            .uri(URI.create("https://api.example.com/users"))
            .header("Content-Type", "application/json")
            .POST(HttpRequest.BodyPublishers.ofString(json))
            .build();
        
        HttpResponse<String> postResponse = client.send(postRequest, 
            HttpResponse.BodyHandlers.ofString());
        
        // WebSocket support (Java 11+)
        WebSocket ws = client.newWebSocketBuilder()
            .buildAsync(URI.create("ws://example.com/socket"), new WebSocket.Listener() {})
            .join();
    }
}

Garbage Collector and Performance

Java 11 → G1GC (default), ZGC experimental
Java 17 → Stable ZGC, G1GC improvements, Shenandoah
Java 11:
- G1GC by default
- ZGC requires experimental flag: -XX:+UnlockExperimentalVMOptions -XX:+UseZGC
- Higher GC pauses
Java 17:
- Stable ZGC: -XX:+UseZGC (no experimental flag)
- Shenandoah GC available: -XX:+UseShenandoahGC
- GC pauses below 10ms even for terabyte heaps
- Compressed oops supported for ZGC
- Better container integration
GC Comparison:
- G1GC: balanced, good for most cases
- ZGC: very low pauses, ideal for large heaps
- Shenandoah: low pauses, concurrent

Removed APIs and Features

Removed between Java 11 and Java 17
// ==================== APIs REMOVED IN JAVA 17 ====================

// 1. Applet API (deprecated since Java 9, removed in Java 17)
// - java.applet.Applet
// - javax.swing.JApplet

// 2. RMI Activation (deprecated in Java 15, removed in Java 17)
// - java.rmi.activation.Activatable
// - java.rmi.activation.ActivationGroup

// 3. Security Manager (@Deprecated(forRemoval=true))
// System.setSecurityManager(...) - deprecated, will be removed

// 4. ConcurrentHashMap - old methods
// ConcurrentHashMap.keySet().map(...) - some variants deprecated

// ==================== APIS DEPRECATED FOR REMOVAL ====================

// Finalizer (deprecated since Java 9)
@Override
@Deprecated(since="9", forRemoval=true)
protected void finalize() {
    // Do not use anymore! Use Cleaner or try-with-resources
}

// Thread.stop(), Thread.suspend(), Thread.resume()
Thread t = new Thread();
t.stop();  // @Deprecated(forRemoval=true) - do not use!

// URL constructor (deprecated for removal)
new URL("http", "example.com", 80, "/path");  // Use URI.toURL() instead

// System.runFinalizersOnExit()
System.runFinalizersOnExit(true);  // Deprecated, dangerous!
Warning: The Security Manager will be completely removed in a future version. Migrate code that uses it.

Migration Guide from Java 11 to Java 17

PhaseRecommended ActionsDetails
1️⃣ Update toolchain Update Maven/Gradle compiler, surefire, failsafe plugins
2️⃣ Check dependencies Ensure all libraries support Java 17
3️⃣ Remove deprecated code Remove calls to finalize(), stop(), suspend()
4️⃣ Test with --illegal-access=deny Identify accesses to internal APIs
5️⃣ Adopt new features Records, Switch expressions, Text blocks, Pattern matching
