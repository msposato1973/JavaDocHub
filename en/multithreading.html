Multithreading and Concurrency in Java

Thread, Runnable, synchronization, ExecutorService, Concurrent Collections, and best practices

What is Multithreading?

Multithreading is the ability of a program to execute multiple threads (sub-processes) simultaneously. Java natively supports multithreading, allowing you to fully utilize multi-core processors and create more responsive applications.
Benefits of multithreading:
- Performance: leverages multi-core processors
- Responsiveness: UI doesn't freeze during long operations
- Resource sharing: threads share memory
- Scalability: multiple requests handled simultaneously

Creating Threads

Three main ways to create threads

📌 METHOD 1: Extend Thread

class ExtendedThread extends Thread {
    private String name;
    
    public ExtendedThread(String name) {
        this.name = name;
    }
    
    @Override
    public void run() {
        for (int i = 1; i <= 5; i++) {
            System.out.println(name + " - step " + i);
            try {
                Thread.sleep(500);
            } catch (InterruptedException e) {
                System.out.println(name + " interrupted");
            }
        }
    }
}

// Usage
Thread t1 = new ExtendedThread("Thread-1");
Thread t2 = new ExtendedThread("Thread-2");
t1.start();
t2.start();

📌 METHOD 2: Implement Runnable (recommended)

class RunnableImpl implements Runnable {
    private String name;
    
    public RunnableImpl(String name) {
        this.name = name;
    }
    
    @Override
    public void run() {
        for (int i = 1; i <= 5; i++) {
            System.out.println(name + " [Runnable] - " + i);
            try {
                Thread.sleep(400);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
    }
}

// Usage
Thread t3 = new Thread(new RunnableImpl("Runnable-1"));
t3.start();

📌 METHOD 3: Lambda Expression (Java 8+)

Thread t4 = new Thread(() -> {
    for (int i = 1; i <= 3; i++) {
        System.out.println("Lambda: " + i);
    }
});
t4.start();
Note: Implementing Runnable is preferred because Java doesn't support multiple inheritance. It also allows the use of lambda expressions.

Thread Lifecycle

NEW → RUNNABLE → RUNNING → BLOCKED/WAITING/TIMED_WAITING → TERMINATED
public class LifecycleDemo {
    public static void main(String[] args) throws InterruptedException {
        Thread thread = new Thread(() -> {
            System.out.println("Thread running...");
            try {
                Thread.sleep(1000);  // TIMED_WAITING
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println("Thread terminated");
        });
        
        System.out.println("State after creation: " + thread.getState());  // NEW
        
        thread.start();
        System.out.println("State after start: " + thread.getState());  // RUNNABLE
        
        Thread.sleep(500);
        System.out.println("State during sleep: " + thread.getState());  // TIMED_WAITING
        
        thread.join();
        System.out.println("State after join: " + thread.getState());  // TERMINATED
    }
}
Output:
State after creation: NEW
State after start: RUNNABLE
Thread running...
State during sleep: TIMED_WAITING
Thread terminated
State after join: TERMINATED

Synchronization and Race Conditions

Warning: without synchronization, multiple threads can corrupt shared data

📌 EXAMPLE WITHOUT SYNCHRONIZATION (RACE CONDITION)

class UnsynchronizedCounter {
    private int count = 0;
    
    public void increment() {
        count++;  // Not atomic!
    }
    
    public int getCount() {
        return count;
    }
}

// Problem: the final result might be less than 10000!

📌 EXAMPLE WITH SYNCHRONIZATION (CORRECT)

class SynchronizedCounter {
    private int count = 0;
    
    // Synchronized method (only one thread at a time)
    public synchronized void increment() {
        count++;
    }
    
    public synchronized int getCount() {
        return count;
    }
}

class Worker extends Thread {
    private SynchronizedCounter counter;
    
    public Worker(SynchronizedCounter counter) {
        this.counter = counter;
    }
    
    @Override
    public void run() {
        for (int i = 0; i < 1000; i++) {
            counter.increment();
        }
    }
}

public class SynchronizationDemo {
    public static void main(String[] args) throws InterruptedException {
        SynchronizedCounter counter = new SynchronizedCounter();
        Thread[] threads = new Thread[10];
        
        for (int i = 0; i < 10; i++) {
            threads[i] = new Worker(counter);
            threads[i].start();
        }
        
        for (Thread t : threads) {
            t.join();  // Wait for termination
        }
        
        System.out.println("Final count: " + counter.getCount());
        // Should be 10 * 1000 = 10000
    }
}
Output: Final count: 10000
Alternatives to synchronized:
- volatile for simple variables
- AtomicInteger, AtomicLong, etc.
- ReentrantLock for more flexibility

ExecutorService and Thread Pools

Professional thread management
import java.util.concurrent.*;
import java.util.List;
import java.util.ArrayList;

public class ExecutorServiceDemo {
    public static void main(String[] args) throws Exception {
        // 1. FixedThreadPool - pool with fixed number of threads
        ExecutorService fixedPool = Executors.newFixedThreadPool(5);
        
        // 2. CachedThreadPool - creates threads when needed, reuses existing ones
        ExecutorService cachedPool = Executors.newCachedThreadPool();
        
        // 3. SingleThreadExecutor - single sequential thread
        ExecutorService singlePool = Executors.newSingleThreadExecutor();
        
        // 4. ScheduledThreadPool - for scheduled tasks
        ScheduledExecutorService scheduler = Executors.newScheduledThreadPool(2);
        
        // ========== TASK EXECUTION ==========
        // Execute task without return value
        fixedPool.submit(() -> {
            System.out.println("Task executed by: " + Thread.currentThread().getName());
        });
        
        // Execute with result (Callable)
        Callable<Integer> taskCallable = () -> {
            Thread.sleep(1000);
            return 42;
        };
        
        Future<Integer> future = fixedPool.submit(taskCallable);
        System.out.println("Result: " + future.get());
        
        // ========== SCHEDULED TASKS ==========
        // Executes after 2 seconds
        scheduler.schedule(() -> System.out.println("Task after 2 seconds"), 2, TimeUnit.SECONDS);
        
        // Executes periodically every 3 seconds (after 1 second initial delay)
        scheduler.scheduleAtFixedRate(() -> System.out.println("Periodic task"), 1, 3, TimeUnit.SECONDS);
        
        // ========== INVOKE ALL ==========
        List<Callable<String>> tasks = Arrays.asList(
            () -> "Task 1 completed",
            () -> "Task 2 completed",
            () -> "Task 3 completed"
        );
        
        List<Future<String>> results = fixedPool.invokeAll(tasks);
        for (Future<String> res : results) {
            System.out.println(res.get());
        }
        
        // ========== SHUTDOWN ==========
        fixedPool.shutdown();
        try {
            if (!fixedPool.awaitTermination(60, TimeUnit.SECONDS)) {
                fixedPool.shutdownNow();
            }
        } catch (InterruptedException e) {
            fixedPool.shutdownNow();
        }
        
        // scheduler.shutdown();  // Remember to shutdown the scheduler too
    }
}
Types of ExecutorService:
- newFixedThreadPool(n): pool with fixed number of threads
- newCachedThreadPool(): creates threads when needed, reuses existing ones
- newSingleThreadExecutor(): single sequential thread
- newScheduledThreadPool(n): for scheduled tasks

Thread Safety with Concurrent Collections

Thread-safe collections for concurrent environments
import java.util.concurrent.*;
import java.util.*;

public class ConcurrentCollectionsDemo {
    public static void main(String[] args) throws InterruptedException {
        // HashMap is NOT thread-safe!
        // Map<String, Integer> map = new HashMap<>();  // PROBLEM!
        
        // ========== THREAD-SAFE SOLUTIONS ==========
        
        // 1. ConcurrentHashMap
        Map<String, Integer> concurrentMap = new ConcurrentHashMap<>();
        
        // 2. CopyOnWriteArrayList (great for frequent reads, rare writes)
        List<Integer> copyOnWriteList = new CopyOnWriteArrayList<>();
        
        // 3. ConcurrentLinkedQueue (non-blocking queue)
        Queue<String> concurrentQueue = new ConcurrentLinkedQueue<>();
        
        // 4. BlockingQueue (blocking queue)
        BlockingQueue<Integer> blockingQueue = new ArrayBlockingQueue<>(10);
        
        // ========== PRODUCER-CONSUMER EXAMPLE ==========
        BlockingQueue<Integer> queue = new LinkedBlockingQueue<>();
        
        // Producer
        Thread producer = new Thread(() -> {
            try {
                for (int i = 1; i <= 10; i++) {
                    queue.put(i);
                    System.out.println("📦 Produced: " + i);
                    Thread.sleep(100);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });
        
        // Consumer
        Thread consumer = new Thread(() -> {
            try {
                while (true) {
                    Integer val = queue.take();
                    System.out.println("📦 Consumed: " + val);
                    if (val == 10) break;
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });
        
        producer.start();
        consumer.start();
        producer.join();
        consumer.join();
        
        // ========== CONCURRENT HASHMAP EXAMPLE ==========
        System.out.println("\n=== ConcurrentHashMap example ===");
        ConcurrentHashMap<String, Integer> grades = new ConcurrentHashMap<>();
        
        // Atomic operations
        grades.put("Alice", 85);
        grades.put("Bob", 70);
        
        // putIfAbsent - inserts only if the key doesn't exist
        grades.putIfAbsent("Alice", 100);  // Does nothing
        grades.putIfAbsent("Diana", 90);   // Inserts
        
        // Conditional replace
        grades.replace("Bob", 70, 75);     // Replaces only if current value is 70
        
        // compute
        grades.compute("Alice", (k, v) -> v + 5);
        
        System.out.println("Grades: " + grades);
    }
}

📦 Consumed: 1
...
📦 Produced: 10
📦 Consumed: 10

wait(), notify(), notifyAll()

Inter-thread communication
class MessageQueue {
    private String message;
    private boolean empty = true;
    
    public synchronized String take() throws InterruptedException {
        while (empty) {
            wait();  // Waits for message to be available
        }
        empty = true;
        notifyAll();  // Notifies producers
        return message;
    }
    
    public synchronized void put(String message) throws InterruptedException {
        while (!empty) {
            wait();  // Waits for message to be consumed
        }
        empty = false;
        this.message = message;
        notifyAll();  // Notifies consumers
    }
}

public class WaitNotifyDemo {
    public static void main(String[] args) {
        MessageQueue queue = new MessageQueue();
        
        // Consumer
        Thread consumer = new Thread(() -> {
            try {
                for (int i = 0; i < 5; i++) {
                    String msg = queue.take();
                    System.out.println("📩 Received: " + msg);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });
        
        // Producer
        Thread producer = new Thread(() -> {
            String[] messages = {"Msg1", "Msg2", "Msg3", "Msg4", "Msg5"};
            try {
                for (String msg : messages) {
                    queue.put(msg);
                    System.out.println("📤 Sent: " + msg);
                    Thread.sleep(500);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });
        
        consumer.start();
        producer.start();
    }
}
Output:
📤 Sent: Msg1
📩 Received: Msg1
📤 Sent: Msg2
📩 Received: Msg2
...
Important: wait(), notify(), and notifyAll() must be called only within synchronized blocks!

CompletableFuture (Asynchronous Programming)

Java 8+ - Non-blocking asynchronous pipelines
import java.util.concurrent.*;
import java.util.List;

public class CompletableFutureDemo {
    public static void main(String[] args) throws Exception {
        // ========== SIMPLE ASYNCHRONOUS EXECUTION ==========
        CompletableFuture<String> future = CompletableFuture.supplyAsync(() -> {
            try { Thread.sleep(1000); } catch (InterruptedException e) {}
            return "Result";
        });
        
        future.thenAccept(result -> System.out.println("Received: " + result));
        
        // ========== PIPELINE OF OPERATIONS ==========
        CompletableFuture.supplyAsync(() -> "Hello")
            .thenApply(s -> s + " World")
            .thenApply(String::toUpperCase)
            .thenAccept(System.out::println);
        
        // ========== COMBINING MULTIPLE FUTURES ==========
        CompletableFuture<Integer> future1 = CompletableFuture.supplyAsync(() -> 10);
        CompletableFuture<Integer> future2 = CompletableFuture.supplyAsync(() -> 20);
        
        CompletableFuture<Integer> sum = future1.thenCombine(future2, (a, b) -> a + b);
        System.out.println("Sum: " + sum.get());
        
        // ========== ERROR HANDLING ==========
        CompletableFuture.supplyAsync(() -> {
            if (true) throw new RuntimeException("Error!");
            return "OK";
        }).exceptionally(ex -> {
            System.out.println("Error: " + ex.getMessage());
            return "Fallback";
        }).thenAccept(System.out::println);
        
        // ========== WAITING FOR ALL FUTURES ==========
        List<CompletableFuture<String>> futures = List.of(
            CompletableFuture.supplyAsync(() -> "Task 1"),
            CompletableFuture.supplyAsync(() -> "Task 2"),
            CompletableFuture.supplyAsync(() -> "Task 3")
        );
        
        CompletableFuture.allOf(futures.toArray(new CompletableFuture[0]))
            .thenRun(() -> System.out.println("All tasks completed"));
        
        // ========== RACE BETWEEN FUTURES ==========
        CompletableFuture<String> fast = CompletableFuture.supplyAsync(() -> {
            sleep(100); return "Fast";
        });
        CompletableFuture<String> slow = CompletableFuture.supplyAsync(() -> {
            sleep(500); return "Slow";
        });
        
        CompletableFuture<Object> first = CompletableFuture.anyOf(fast, slow);
        System.out.println("First completed: " + first.get());
    }
    
    private static void sleep(int ms) {
        try { Thread.sleep(ms); } catch (InterruptedException e) {}
    }
}
Output:
HELLO WORLD
Sum: 30
Error: java.lang.RuntimeException: Error!
Fallback
Received: Result
All tasks completed
First completed: Fast

Lock and ReentrantLock

More flexible alternative to synchronized
import java.util.concurrent.locks.*;
import java.util.*;

public class LockDemo {
    
    // ========== REENTRANT LOCK ==========
    static class CounterWithLock {
        private int count = 0;
        private final Lock lock = new ReentrantLock();
        
        public void increment() {
            lock.lock();
            try {
                count++;
            } finally {
                lock.unlock();  // Important: always in finally!
            }
        }
        
        public int getCount() {
            lock.lock();
            try {
                return count;
            } finally {
                lock.unlock();
            }
        }
        
        // TryLock - non-blocking
        public boolean tryIncrement() {
            if (lock.tryLock()) {
                try {
                    count++;
                    return true;
                } finally {
                    lock.unlock();
                }
            }
            return false;
        }
        
        // TryLock with timeout
        public boolean tryIncrementWithTimeout(long timeout, TimeUnit unit) throws InterruptedException {
            if (lock.tryLock(timeout, unit)) {
                try {
                    count++;
                    return true;
                } finally {
                    lock.unlock();
                }
            }
            return false;
        }
    }
    
    // ========== READ WRITE LOCK ==========
    static class Cache {
        private final Map<String, String> data = new HashMap<>();
        private final ReadWriteLock readWriteLock = new ReentrantReadWriteLock();
        private final Lock readLock = readWriteLock.readLock();
        private final Lock writeLock = readWriteLock.writeLock();
        
        public String get(String key) {
            readLock.lock();
            try {
                return data.get(key);
            } finally {
                readLock.unlock();
            }
        }
        
        public void put(String key, String value) {
            writeLock.lock();
            try {
                data.put(key, value);
                System.out.println("Inserted: " + key + "=" + value);
            } finally {
                writeLock.unlock();
            }
        }
        
        public void clear() {
            writeLock.lock();
            try {
                data.clear();
                System.out.println("Cache cleared");
            } finally {
                writeLock.unlock();
            }
        }
    }
    
    public static void main(String[] args) throws InterruptedException {
        // Test ReentrantLock
        CounterWithLock counter = new CounterWithLock();
        
        Thread[] threads = new Thread[10];
        for (int i = 0; i < 10; i++) {
            threads[i] = new Thread(() -> {
                for (int j = 0; j < 1000; j++) {
                    counter.increment();
                }
            });
            threads[i].start();
        }
        
        for (Thread t : threads) {
            t.join();
        }
        System.out.println("Counter with Lock: " + counter.getCount());
        
        // Test ReadWriteLock
        Cache cache = new Cache();
        
        // Writers
        Thread writer = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                cache.put("key" + i, "value" + i);
            }
        });
        
        // Readers (can be concurrent)
        Thread reader1 = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                System.out.println("Reader1: " + cache.get("key" + i));
            }
        });
        
        Thread reader2 = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                System.out.println("Reader2: " + cache.get("key" + i));
            }
        });
        
        writer.start();
        writer.join();
        reader1.start();
        reader2.start();
    }
}
Output:
Counter with Lock: 10000
Inserted: key0=value0
...
Reader1: value0
Reader2: value0
...
Advantages of Lock over synchronized:
- tryLock() non-blocking
- lockInterruptibly() interruptible
- ReadWriteLock to optimize concurrent reads
- More control over timeouts

Atomic Classes (java.util.concurrent.atomic)

Atomic operations without explicit synchronization
import java.util.concurrent.atomic.*;
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class AtomicDemo {
    public static void main(String[] args) throws InterruptedException {
        // ========== ATOMIC INTEGER ==========
        AtomicInteger atomicInt = new AtomicInteger(0);
        
        System.out.println("=== AtomicInteger ===");
        System.out.println("Initial value: " + atomicInt.get());
        
        // Atomic operations
        atomicInt.incrementAndGet();      // 1
        atomicInt.addAndGet(5);           // 6
        atomicInt.compareAndSet(6, 10);   // CAS (Compare And Swap)
        
        System.out.println("After operations: " + atomicInt.get());
        
        // getAndUpdate
        atomicInt.getAndUpdate(n -> n * 2);
        System.out.println("After update: " + atomicInt.get());
        
        // ========== ATOMIC BOOLEAN ==========
        AtomicBoolean flag = new AtomicBoolean(false);
        boolean wasFalse = flag.compareAndSet(false, true);
        System.out.println("\n=== AtomicBoolean ===");
        System.out.println("CompareAndSet succeeded? " + wasFalse);
        System.out.println("Flag value: " + flag.get());
        
        // ========== ATOMIC REFERENCE ==========
        AtomicReference<String> ref = new AtomicReference<>("init");
        ref.set("new value");
        ref.compareAndSet("new value", "updated");
        System.out.println("\n=== AtomicReference ===");
        System.out.println("Reference value: " + ref.get());
        
        // ========== PRACTICAL EXAMPLE: LOCK-FREE COUNTER ==========
        System.out.println("\n=== Counter with AtomicLong ===");
        AtomicLong counter = new AtomicLong(0);
        
        ExecutorService executor = Executors.newFixedThreadPool(10);
        for (int i = 0; i < 1000; i++) {
            executor.submit(counter::incrementAndGet);
        }
        
        executor.shutdown();
        Thread.sleep(1000);
        System.out.println("Atomic counter: " + counter.get());  // 1000
        
        // ========== LONG ADDER (better for high concurrency) ==========
        System.out.println("\n=== LongAdder (high concurrency) ===");
        LongAdder adder = new LongAdder();
        
        // 1000 threads incrementing
        ExecutorService executor2 = Executors.newFixedThreadPool(10);
        for (int i = 0; i < 1000; i++) {
            executor2.submit(() -> adder.increment());
        }
        
        executor2.shutdown();
        Thread.sleep(1000);
        System.out.println("LongAdder sum: " + adder.sum());
        System.out.println("LongAdder longValue: " + adder.longValue());
        
        // ========== ATOMIC INTEGER ARRAY ==========
        System.out.println("\n=== AtomicIntegerArray ===");
        AtomicIntegerArray array = new AtomicIntegerArray(5);
        array.set(0, 10);
        array.set(1, 20);
        array.incrementAndGet(0);
        array.addAndGet(1, 5);
        System.out.println("Array[0]: " + array.get(0));
        System.out.println("Array[1]: " + array.get(1));
    }
}

Best Practices for Multithreading

Fundamental rules for writing safe concurrent code
// ==================== 1. PREFER Runnable OVER Thread ====================
// ✅ Good practice
new Thread(() -> {
    // task
}).start();

// ==================== 2. USE ExecutorService INSTEAD OF MANUAL THREADS ====================
ExecutorService executor = Executors.newFixedThreadPool(10);
executor.submit(() -> { /* task */ });
executor.shutdown();

// ==================== 3. MAKE SHARED OBJECTS IMMUTABLE ====================
public final class ImmutablePerson {
    private final String name;
    private final int age;
    
    public ImmutablePerson(String name, int age) {
        this.name = name;
        this.age = age;
    }
    public String getName() { return name; }
    public int getAge() { return age; }
}

// ==================== 4. USE CONCURRENT COLLECTIONS ====================
// ❌ Map<String, String> map = new HashMap<>();
// ✅ Map<String, String> map = new ConcurrentHashMap<>();

// ==================== 5. MINIMIZE SYNCHRONIZATION ====================
// Synchronize only critical sections, not entire methods
public void increment() {
    synchronized(this) {
        // only the critical part
        counter++;
    }
}

// ==================== 6. USE volatile FOR SIMPLE FLAGS ====================
private volatile boolean running = true;

// ==================== 7. HANDLE THREAD INTERRUPTION ====================
while (!Thread.currentThread().isInterrupted()) {
    try {
        // work
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();  // Restores the status
        break;
    }
}

// ==================== 8. AVOID DEADLOCKS ====================
// - Always acquire locks in the same order
// - Use tryLock with timeouts
// - Minimize nested locks

// ==================== 9. USE THREAD LOCAL WHEN APPROPRIATE ====================
public class DateFormatter {
    private static final ThreadLocal<SimpleDateFormat> dateFormat = 
        ThreadLocal.withInitial(() -> new SimpleDateFormat("yyyy-MM-dd"));
    
    public String format(Date date) {
        return dateFormat.get().format(date);
    }
}

// ==================== 10. USE COMPLETABLE FUTURE FOR ASYNCHRONOUS OPERATIONS ====================
CompletableFuture.supplyAsync(() -> "Hello")
    .thenApply(String::toUpperCase)
    .thenAccept(System.out::println);
Beware of Deadlocks: Ensure threads acquire locks in the same order to avoid deadlocks!
Concurrent code checklist:
✓ Use ExecutorService instead of manual threads
✓ Prefer atomic classes and concurrent collections
✓ Make shared objects immutable when possible
✓ Use volatile for simple flags
✓ Handle InterruptedException correctly
✓ Avoid deadlocks by ordering lock acquisition
✓ Use CompletableFuture for asynchronous operations
