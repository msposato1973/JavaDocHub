Exception Handling in Java

try-catch-finally, throw, throws, custom exceptions, try-with-resources, and best practices

What is an Exception?

An exception in Java is an event that disrupts the normal flow of a program's execution. Java provides a robust mechanism for handling errors through exceptions, allowing separation of business logic from error handling.
Advantages of exception handling:
- Separation of error logic from business logic
- Error propagation along the call stack
- Grouping of error types
- Centralized error handling

Exception Hierarchy

Throwable is the base class of all exceptions
// Exception hierarchy in Java:
//
//                    Throwable
//                   /        \
//              Error        Exception
//              /    \       /        \
//   OutOfMemoryError  StackOverflowError  RuntimeException  IOException
//                                         /        \
//                              NullPointerException  IllegalArgumentException
//
// - Error: serious environment errors (non-recoverable)
// - Exception (checked): require handling (try-catch or throws)
// - RuntimeException (unchecked): do not require handling

public class ExceptionHierarchy {
    public static void main(String[] args) {
        // RuntimeException (does not require try-catch)
        String s = null;
        try {
            s.length();  // NullPointerException
        } catch (NullPointerException e) {
            System.out.println("NPE handled: " + e.getMessage());
        }
        
        // Checked Exception (requires try-catch or throws)
        try {
            FileReader file = new FileReader("file.txt");
        } catch (FileNotFoundException e) {
            System.out.println("File not found!");
        }
    }
}
Note: Checked exceptions (like IOException) MUST be handled with try-catch or declared with throws. Unchecked (RuntimeException) are optional but recommended.

try-catch-finally

Fundamental blocks for exception handling
public class TryCatchFinallyDemo {
    public static void main(String[] args) {
        System.out.println(safeDivision(10, 2));
        System.out.println(safeDivision(10, 0));
    }
    
    public static int safeDivision(int a, int b) {
        try {
            System.out.println("Attempting division: " + a + "/" + b);
            return a / b;
        } catch (ArithmeticException e) {
            System.err.println("ERROR: " + e.getMessage());
            return -1;
        } finally {
            System.out.println("Finally - always executed");
        }
    }
}
Output:
Attempting division: 10/2
Finally - always executed
5
Attempting division: 10/0
ERROR: / by zero
Finally - always executed
-1
Important: The finally block is executed ALWAYS, even if there is a return in the try or catch. It is ideal for closing resources.

throw and throws

throw → throws an exception | throws → declares that a method may throw exceptions
public class ThrowThrowsDemo {
    
    // throws declares that the method may throw an exception
    public static void validateAge(int age) throws IllegalArgumentException {
        if (age < 0 || age > 150) {
            // throw actually throws the exception
            throw new IllegalArgumentException("Invalid age: " + age);
        }
        System.out.println("Valid age: " + age);
    }
    
    // Multiple throws
    public static void readFile(String path) throws FileNotFoundException, IOException {
        FileReader reader = new FileReader(path);
        int data = reader.read();
        reader.close();
    }
    
    // Rethrow exception with cause
    public static void processData() throws BusinessException {
        try {
            // operation that may fail
            readFile("data.txt");
        } catch (IOException e) {
            // Preserves original cause
            throw new BusinessException("Data processing error", e);
        }
    }
    
    public static void main(String[] args) {
        try {
            validateAge(200);  // Throws exception
        } catch (IllegalArgumentException e) {
            System.out.println("Caught: " + e.getMessage());
        }
        
        try {
            validateAge(25);   // Ok
        } catch (IllegalArgumentException e) {
            System.out.println(e.getMessage());
        }
        
        // Handling multiple exceptions in a single catch (Java 7+)
        try {
            readFile("test.txt");
        } catch (FileNotFoundException | IOException e) {
            System.out.println("I/O Error: " + e.getMessage());
        }
    }
}

class BusinessException extends Exception {
    public BusinessException(String message, Throwable cause) {
        super(message, cause);
    }
}
Output:
Caught: Invalid age: 200
Valid age: 25

Custom Exceptions

Create your own exceptions for domain-specific errors
// ==================== UNCHECKED EXCEPTION ====================
// Extends RuntimeException (does not require try-catch)
class InsufficientBalanceException extends RuntimeException {
    private final double currentBalance;
    private final double requestedAmount;
    
    public InsufficientBalanceException(double currentBalance, double requestedAmount) {
        super(String.format("Insufficient balance: €%.2f available, €%.2f requested", 
              currentBalance, requestedAmount));
        this.currentBalance = currentBalance;
        this.requestedAmount = requestedAmount;
    }
    
    public double getCurrentBalance() { return currentBalance; }
    public double getRequestedAmount() { return requestedAmount; }
}

// ==================== CHECKED EXCEPTION ====================
// Extends Exception (requires try-catch or throws)
class AccountNotFoundException extends Exception {
    private final String iban;
    private final LocalDateTime timestamp;
    
    public AccountNotFoundException(String iban) {
        super("Account with IBAN " + iban + " not found in the system");
        this.iban = iban;
        this.timestamp = LocalDateTime.now();
    }
    
    public AccountNotFoundException(String iban, Throwable cause) {
        super("Account with IBAN " + iban + " not found in the system", cause);
        this.iban = iban;
        this.timestamp = LocalDateTime.now();
    }
    
    public String getIban() { return iban; }
    public LocalDateTime getTimestamp() { return timestamp; }
}

// ==================== BANK ACCOUNT CLASS ====================
class BankAccount {
    private double balance;
    private String iban;
    private String owner;
    
    public BankAccount(String iban, String owner, double initialBalance) {
        this.iban = iban;
        this.owner = owner;
        this.balance = initialBalance;
    }
    
    public void withdraw(double amount) {
        if (amount > balance) {
            throw new InsufficientBalanceException(balance, amount);
        }
        balance -= amount;
        System.out.printf("Withdrawal of €%.2f completed. New balance: €%.2f%n", amount, balance);
    }
    
    public void deposit(double amount) {
        if (amount <= 0) {
            throw new IllegalArgumentException("Invalid deposit amount: " + amount);
        }
        balance += amount;
        System.out.printf("Deposit of €%.2f completed. New balance: €%.2f%n", amount, balance);
    }
    
    public static BankAccount findByIban(String iban) throws AccountNotFoundException {
        // Simulate search
        if (!iban.equals("IT1234567890")) {
            throw new AccountNotFoundException(iban);
        }
        return new BankAccount(iban, "Mario Rossi", 1000.0);
    }
    
    public double getBalance() { return balance; }
    public String getIban() { return iban; }
    public String getOwner() { return owner; }
}

    public static void main(String[] args) {
        BankAccount account = new BankAccount("IT1234567890", "Mario Rossi", 1000.0);

try-with-resources (AutoCloseable)

Java 7+ - Automatic resource closing

📌 OLD WAY (Java 6 and earlier)

// Without try-with-resources - so much boilerplate!
BufferedReader br = null;
try {
    br = new BufferedReader(new FileReader("file.txt"));
    String line = br.readLine();
    System.out.println(line);
} catch (IOException e) {
    e.printStackTrace();
} finally {
    if (br != null) {
        try {
            br.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

📌 NEW WAY (try-with-resources)

// With try-with-resources - much cleaner!
try (BufferedReader br = new BufferedReader(new FileReader("file.txt"))) {
    String line = br.readLine();
    System.out.println(line);
} catch (IOException e) {
    System.err.println("Read error: " + e.getMessage());
}
// br is automatically closed!

📌 MULTIPLE RESOURCES

// Managing multiple resources simultaneously
try (FileInputStream fis = new FileInputStream("input.dat");
     FileOutputStream fos = new FileOutputStream("output.dat");
     BufferedReader reader = new BufferedReader(new FileReader("log.txt"))) {
    
    int data;
    while ((data = fis.read()) != -1) {
        fos.write(data);
    }
    System.out.println("Copy completed");
    
} catch (IOException e) {
    e.printStackTrace();
}
// All three resources are automatically closed!

📌 CUSTOM AUTO-CLOSEABLE CLASS

class CustomResource implements AutoCloseable {
    private String name;
    
    public CustomResource(String name) {
        this.name = name;
        System.out.println("🔓 Opening resource: " + name);
    }
    
    public void doSomething() {
        System.out.println("⚙️ Using resource: " + name);
    }
    
    @Override
    public void close() throws Exception {
        System.out.println("🔒 Closing resource: " + name);
    }
}

// Usage
try (CustomResource res1 = new CustomResource("DB Connection");
     CustomResource res2 = new CustomResource("File Handler")) {
    
    res1.doSomething();
    res2.doSomething();
    
} catch (Exception e) {
    e.printStackTrace();
}
// Output:
// 🔓 Opening resource: DB Connection
// 🔓 Opening resource: File Handler
// ⚙️ Using resource: DB Connection
// ⚙️ Using resource: File Handler
// 🔒 Closing resource: File Handler
// 🔒 Closing resource: DB Connection
Advantages of try-with-resources:
- Cleaner and more readable code
- Automatic closing even in case of exceptions
- Support for multiple resources
- Resources are closed in reverse order of opening

Best Practices for Exceptions

Fundamental rules for writing robust code
// ==================== 1. NEVER IGNORE EXCEPTIONS ====================
// ❌ WRONG
try {
    // code
} catch (Exception e) {
    // Do nothing - very bad practice!
}

// ✅ CORRECT - at least log the error
try {
    // code
} catch (Exception e) {
    logger.error("Error during operation", e);
    // or e.printStackTrace();
}

// ==================== 2. USE SPECIFIC EXCEPTIONS ====================
// ❌ WRONG - too generic
try {
    // code
} catch (Exception e) {
    // catches everything
}

// ✅ CORRECT - specific exceptions
try {
    // code
} catch (FileNotFoundException e) {
    System.out.println("File not found");
} catch (IOException e) {
    System.out.println("I/O error");
} catch (SQLException e) {
    System.out.println("Database error");
}

// ==================== 3. PROPAGATE EXCEPTIONS WHEN APPROPRIATE ====================
// ✅ When the calling method needs to know about the error
public void readConfiguration() throws IOException {
    // if it fails, the caller decides how to handle
}

// ==================== 4. USE FINALLY FOR CLEANUP (OR TRY-WITH-RESOURCES) ====================
Connection conn = null;
try {
    conn = dataSource.getConnection();
    // use connection
} finally {
    if (conn != null) {
        try {
            conn.close();
        } catch (SQLException e) {
            // log but don't rethrow
        }
    }
}

// ==================== 5. DON'T USE EXCEPTIONS FOR NORMAL FLOW ====================
// ❌ WRONG
try {
    return array[index];
} catch (IndexOutOfBoundsException e) {
    return null;  // don't use exceptions for logical checks!
}

// ✅ CORRECT
if (index >= 0 && index < array.length) {
    return array[index];
} else {
    return null;
}

// ==================== 6. PROVIDE MEANINGFUL MESSAGES ====================
// ❌ WRONG
throw new RuntimeException("Error");

// ✅ CORRECT
throw new IllegalArgumentException(
    String.format("Value %d is invalid. Must be between %d and %d", 
                  value, min, max)
);

// ==================== 7. PRESERVE THE ORIGINAL STACK TRACE ====================
// ❌ WRONG - loses the original cause
try {
    // operation
} catch (SQLException e) {
    throw new MyBusinessException("DB Error");
}

// ✅ CORRECT - preserves the cause
try {
    // operation
} catch (SQLException e) {
    throw new MyBusinessException("DB Error", e);  // passes the cause
}

// ==================== 8. DOCUMENT EXCEPTIONS ====================
/**
 * Calculates the square root of a number.
 * 
 * @param number the number to calculate the square root of
 * @return the square root of the number
 * @throws IllegalArgumentException if the number is negative
 */
public double squareRoot(double number) throws IllegalArgumentException {
    if (number < 0) {
        throw new IllegalArgumentException("Negative number: " + number);
    }
    return Math.sqrt(number);
}
Checklist for quality exceptions:
✓ Use specific exceptions (not generic Exception)
✓ Provide informative error messages
✓ Never hide exceptions (empty catch)
✓ Use try-with-resources for AutoCloseable resources
✓ Document exceptions with @throws in JavaDoc
✓ Preserve the original stack trace (cause)

📌 EXAMPLE 1: User input validation

import java.util.Scanner;

public class InputValidator {
    
    public static int safeParseInt(String input) {
        try {
            return Integer.parseInt(input);
        } catch (NumberFormatException e) {
            System.err.println("Invalid input: '" + input + "' is not a number");
            return 0;  // default value
        }
    }
    
    public static Optional<Integer> parseOptionalInt(String input) {
        try {
            return Optional.of(Integer.parseInt(input));
        } catch (NumberFormatException e) {
            return Optional.empty();
        }
    }
    
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        
        System.out.print("Enter a number: ");
        String input = scanner.nextLine();
        
        // Method 1: with default value
        int number1 = safeParseInt(input);
        System.out.println("Number (default): " + number1);
        
        // Method 2: with Optional
        Optional<Integer> number2 = parseOptionalInt(input);
        number2.ifPresentOrElse(
            n -> System.out.println("Number (optional): " + n),
            () -> System.out.println("Invalid input, no number")
        );
        
        scanner.close();
    }
}

📌 EXAMPLE 2: File reading with multiple exceptions

import java.io.*;
import java.nio.file.*;
import java.util.*;

public class FileProcessor {
    
    public static List<String> readLines(String path) {
        List<String> lines = new ArrayList<>();
        
        try (BufferedReader reader = new BufferedReader(new FileReader(path))) {
            String line;
            while ((line = reader.readLine()) != null) {
                lines.add(line);
            }
        } catch (FileNotFoundException e) {
            System.err.println("File not found: " + path);
            // Create default file
            createDefaultFile(path);
            // Try reading again
            return readLines(path);
        } catch (IOException e) {
            System.err.println("Error during reading: " + e.getMessage());
        }
        
        return lines;
    }
    
    private static void createDefaultFile(String path) {
        try (FileWriter writer = new FileWriter(path)) {
            writer.write("# File created automatically\n");
            writer.write("line1\n");
            writer.write("line2\n");
            writer.write("line3\n");
            System.out.println("Default file created: " + path);
        } catch (IOException e) {
            System.err.println("Unable to create default file: " + e.getMessage());
        }
    }
    
    public static void main(String[] args) {
        List<String> lines = readLines("config.txt");
        System.out.println("Lines read: " + lines.size());
        lines.forEach(System.out::println);
    }
}

import java.time.LocalDateTime;
import java.util.*;

// ==================== RESPONSE MODELS ====================
class ErrorResponse {
    private String code;
    private String message;
    private LocalDateTime timestamp;
    private Map<String, String> details;
    
    public ErrorResponse(String code, String message) {
        this.code = code;
        this.message = message;
        this.timestamp = LocalDateTime.now();
        this.details = new HashMap<>();
    }
    
    public ErrorResponse addDetail(String key, String value) {
        details.put(key, value);
        return this;
    }
    
    // Getters
    public String getCode() { return code; }
    public String getMessage() { return message; }
    public LocalDateTime getTimestamp() { return timestamp; }
    public Map<String, String> getDetails() { return details; }
    
    @Override
    public String toString() {
        return String.format("ErrorResponse{code='%s', message='%s', timestamp=%s, details=%s}",
            code, message, timestamp, details);
    }
}

// ==================== CUSTOM EXCEPTIONS ====================
class ResourceNotFoundException extends RuntimeException {
    private final String resourceType;
    private final String resourceId;
    
    public ResourceNotFoundException(String resourceType, String resourceId) {
        super(String.format("%s with ID '%s' not found", resourceType, resourceId));
        this.resourceType = resourceType;
        this.resourceId = resourceId;
    }
    
    public String getResourceType() { return resourceType; }
    public String getResourceId() { return resourceId; }
}

class ValidationException extends RuntimeException {
    private final Map<String, String> errors;
    
    public ValidationException(Map<String, String> errors) {
        super("Validation error");
        this.errors = errors;
    }
    
    public Map<String, String> getErrors() { return errors; }
}

// ==================== CENTRALIZED EXCEPTION HANDLER ====================
class GlobalExceptionHandler {
    
    public ErrorResponse handleNotFound(ResourceNotFoundException e) {
        return new ErrorResponse("NOT_FOUND", e.getMessage())
            .addDetail("resourceType", e.getResourceType())
            .addDetail("resourceId", e.getResourceId());
    }
    
    public ErrorResponse handleValidation(ValidationException e) {
        ErrorResponse response = new ErrorResponse("VALIDATION_ERROR", e.getMessage());
        e.getErrors().forEach(response::addDetail);
        return response;
    }
    
    public ErrorResponse handleGeneric(Exception e) {
        return new ErrorResponse("INTERNAL_ERROR", "Internal server error")
            .addDetail("exception", e.getClass().getSimpleName());
    }
}

// ==================== USAGE EXAMPLE ====================
public class ApiExceptionDemo {
    
    private static final GlobalExceptionHandler exceptionHandler = new GlobalExceptionHandler();
    
    public static void main(String[] args) {
        // Simulate an API call
        System.out.println("=== Test ResourceNotFoundException ===\n");
        try {
            findUser("999");
        } catch (ResourceNotFoundException e) {
            ErrorResponse response = exceptionHandler.handleNotFound(e);
            System.out.println(response);
        }
        
        System.out.println("\n=== Test ValidationException ===\n");
        try {
            createUser(null, "email");
        } catch (ValidationException e) {
            ErrorResponse response = exceptionHandler.handleValidation(e);
            System.out.println(response);
        }
        
        System.out.println("\n=== Test generic exception ===\n");
        try {
            riskyOperation();
        } catch (Exception e) {
            ErrorResponse response = exceptionHandler.handleGeneric(e);
            System.out.println(response);
        }
    }
    
    static User findUser(String id) {
        if (!id.equals("123")) {
            throw new ResourceNotFoundException("User", id);
        }
        return new User(id, "Mario Rossi", "mario@email.com");
    }
    
    static void createUser(String name, String email) {
        Map<String, String> errors = new HashMap<>();
        
        if (name == null || name.isEmpty()) {
            errors.put("name", "Name is required");
        }
        if (email == null || !email.contains("@")) {
            errors.put("email", "Invalid email");
        }
        
        if (!errors.isEmpty()) {
            throw new ValidationException(errors);
        }
        
        System.out.println("User created: " + name);
    }
    
    static void riskyOperation() {
        throw new RuntimeException("Unexpected database error");
    }
    
    static class User {
        String id, name, email;
        User(String id, String name, String email) {
            this.id = id; this.name = name; this.email = email;
        }
    }
}
