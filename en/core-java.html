Core Java

The fundamentals of the Java language: from JVM to basic syntax, data types, operators, and control structures

What is Java?

Java is an object-oriented, platform-independent (thanks to the JVM), and strongly typed programming language. Created by Sun Microsystems in 1995, it is now one of the most widely used languages in the world for enterprise, mobile (Android), and web applications.
Main features of Java:
- Platform Independent: "Write Once, Run Anywhere" (WORA)
- Object-Oriented: everything is organized into classes and objects
- Robust: automatic memory management (garbage collection)
- Secure: no explicit pointers, sandbox security
- Multithreaded: native support for concurrent programming

JVM, JRE, JDK

Java Virtual Machine, Java Runtime Environment, Java Development Kit

📌 JVM (Java Virtual Machine)

The JVM is the engine that executes Java bytecode. It provides the platform-independent runtime environment.

📌 JRE (Java Runtime Environment)

The JRE includes the JVM and core libraries needed to run Java applications (but not to develop them).

📌 JDK (Java Development Kit)

The JDK includes the JRE plus development tools such as the compiler (javac), debugger, etc.

Basic Syntax

Structure of a Java program
// Single line comment

/*
 * Multi-line comment
 * (used for JavaDoc)
 */

/**
 * JavaDoc documentation
 * @author Author Name
 * @version 1.0
 */
public class ClassName {
    
    // Class variable (static)
    private static int counter = 0;
    
    // Instance variable
    private String name;
    
    // Constructor
    public ClassName(String name) {
        this.name = name;
        counter++;
    }
    
    // Main method (entry point)
    public static void main(String[] args) {
        System.out.println("Hello, World!");
        
        // Creating an object
        ClassName obj = new ClassName("Example");
        
        // Variable declaration
        int number = 42;
        String text = "Java";
        
        // Formatted output
        System.out.printf("Number: %d, Text: %s%n", number, text);
    }
}
Output:
Hello, World!
Number: 42, Text: Java

Operators

Arithmetic, relational, logical, bitwise, assignment

📌 Arithmetic Operators

int a = 10, b = 3;
System.out.println("Sum: " + (a + b));       // 13
System.out.println("Difference: " + (a - b));  // 7
System.out.println("Product: " + (a * b));    // 30
System.out.println("Division: " + (a / b));   // 3 (integer division)
System.out.println("Modulus: " + (a % b));      // 1 (remainder)

// Increment/Decrement
int x = 5;
System.out.println(x++);  // 5 (post-increment)
System.out.println(++x);  // 7 (pre-increment)

📌 Relational Operators

int a = 10, b = 20;
System.out.println(a == b);  // false (equal)
System.out.println(a != b);  // true  (not equal)
System.out.println(a > b);   // false (greater than)
System.out.println(a < b);   // true  (less than)
System.out.println(a >= b);  // false (greater than or equal)
System.out.println(a <= b);  // true  (less than or equal)

📌 Logical Operators

boolean p = true, q = false;
System.out.println(p && q);  // false (logical AND)
System.out.println(p || q);  // true  (logical OR)
System.out.println(!p);      // false (logical NOT)

// Short-circuit evaluation
boolean result = (a > 0) && (b / a > 10);  // b/a not evaluated if a<=0

📌 Ternary Operator

int max = (a > b) ? a : b;  // if a>b returns a, otherwise b

String type = switch (day) {

📌 Loops

// for loop
for (int i = 0; i < 5; i++) {
    System.out.print(i + " ");
}
// Output: 0 1 2 3 4

// while loop
int i = 0;
while (i < 5) {
    System.out.print(i + " ");
    i++;
}

// do-while (executes at least once)
int j = 0;
do {
    System.out.print(j + " ");
    j++;
} while (j < 5);

// enhanced for (for-each)
int[] numbers = {1, 2, 3, 4, 5};
for (int n : numbers) {
    System.out.print(n + " ");
}

// break and continue
for (int k = 0; k < 10; k++) {
    if (k == 5) break;      // exits the loop
    if (k % 2 == 0) continue; // skips even numbers
    System.out.print(k + " ");
}

    {4, 5, 6},
    {7, 8, 9}

for (int i = 0; i < matrix.length; i++) {

Strings

Immutable String, mutable StringBuilder
// ========== CREATION ==========
String s1 = "Hello";
String s2 = new String("World");

// ========== MAIN METHODS ==========
String text = "Java Programming";

System.out.println("Length: " + text.length());           // 16
System.out.println("Uppercase: " + text.toUpperCase());      // JAVA PROGRAMMING
System.out.println("Lowercase: " + text.toLowerCase());      // java programming
System.out.println("Substring: " + text.substring(5, 11));   // Progra
System.out.println("Contains 'Java'? " + text.contains("Java")); // true
System.out.println("Starts with 'Java'? " + text.startsWith("Java")); // true
System.out.println("Index of 'g': " + text.indexOf('g'));   // 10

// ========== COMPARISON (use equals, not ==) ==========
String str1 = "test";
String str2 = new String("test");
System.out.println(str1 == str2);        // false
System.out.println(str1.equals(str2));   // true

// ========== CONCATENATION ==========
String concatenated = "Hello" + " " + "World";
String formatted = String.format("Number: %d, Text: %s", 42, "Java");

// ========== SPLIT AND JOIN ==========
String csv = "apple,pear,banana";
String[] fruits = csv.split(",");
String rejoined = String.join("-", fruits);  // "apple-pear-banana"

// ========== STRINGBUILDER (mutable, more efficient) ==========
StringBuilder sb = new StringBuilder();
sb.append("Java");
sb.append(" ");
sb.append("is");
sb.append(" ");
sb.append("awesome!");
System.out.println(sb.toString());

sb.insert(5, "programming ");
sb.reverse();
System.out.println(sb);

Methods

Declaration, parameters, return, overload, varargs
public class MethodsDemo {
    
    // ========== METHOD WITHOUT PARAMETERS AND WITHOUT RETURN ==========
    public static void greet() {
        System.out.println("Hello from Java!");
    }
    
    // ========== METHOD WITH PARAMETERS AND RETURN ==========
    public static int sum(int a, int b) {
        return a + b;
    }
    
    // ========== OVERLOAD (same name, different parameters) ==========
    public static double sum(double a, double b) {
        return a + b;
    }
    
    public static int sum(int a, int b, int c) {
        return a + b + c;
    }
    
    // ========== VARARGS (variable number of arguments) ==========
    public static int sumVarargs(int... numbers) {
        int total = 0;
        for (int n : numbers) {
            total += n;
        }
        return total;
    }
    
    // ========== RECURSIVE METHOD ==========
    public static long factorial(int n) {
        if (n <= 1) return 1;
        return n * factorial(n - 1);
    }
    
    // ========== PARAMETER PASSING ==========
    // Java is pass-by-value (changes to parameters do not affect the original)
    public static void modifyPrimitive(int x) {
        x = 100;  // does not modify the original variable
    }
    
    public static void modifyObject(StringBuilder sb) {
        sb.append(" added");  // modifies the original object
    }
    
    public static void main(String[] args) {
        greet();
        
        System.out.println("Sum 5+3: " + sum(5, 3));
        System.out.println("Sum 5.5+3.2: " + sum(5.5, 3.2));
        System.out.println("Sum 5+3+2: " + sum(5, 3, 2));
        System.out.println("Sum varargs: " + sumVarargs(1, 2, 3, 4, 5));
        System.out.println("Factorial of 5: " + factorial(5));
        
        // Test parameter passing
        int value = 10;
        modifyPrimitive(value);
        System.out.println("Value after method: " + value);  // 10 (unchanged)
        
        StringBuilder sb = new StringBuilder("Test");
        modifyObject(sb);
        System.out.println("StringBuilder after method: " + sb);  // "Test added"
    }
}
Output:
Hello from Java!
Sum 5+3: 8
Sum 5.5+3.2: 8.7
Sum 5+3+2: 10
Sum varargs: 15
Factorial of 5: 120
Value after method: 10
StringBuilder after method: Test added

Console Input/Output

Scanner, System.out, System.err
import java.util.Scanner;

public class InputOutputDemo {
    public static void main(String[] args) {
        // ========== OUTPUT ==========
        System.out.println("Print with println (adds newline)");
        System.out.print("Print with print (no newline)");
        System.out.printf("Formatted print: %d, %s, %.2f%n", 42, "test", 3.14159);
        System.err.println("This is an error message");
        
        // ========== INPUT WITH SCANNER ==========
        Scanner scanner = new Scanner(System.in);
        
        System.out.println("\n--- Enter your data ---");
        
        System.out.print("Enter your name: ");
        String name = scanner.nextLine();
        
        System.out.print("Enter your age: ");
        int age = scanner.nextInt();
        
        scanner.nextLine();  // consume the remaining newline
        
        System.out.print("Enter your city: ");
        String city = scanner.nextLine();
        
        System.out.print("Enter your salary: ");
        double salary = scanner.nextDouble();
        
        System.out.println("\n--- Summary ---");
        System.out.printf("Name: %s%n", name);
        System.out.printf("Age: %d years%n", age);
        System.out.printf("City: %s%n", city);
        System.out.printf("Salary: €%.2f%n", salary);
        
        scanner.close();
    }
}

Best Practices for Core Java

Fundamental rules for writing quality Java code
// ==================== 1. NAMING CONVENTIONS ====================
// Classes: PascalCase (initial capital)
public class ClassName { }

// Methods and variables: camelCase (initial lowercase)
public void methodName() { }
int variableName = 10;

// Constants: UPPER_SNAKE_CASE
public static final int MAX_VALUE = 100;

// Packages: lowercase (all lowercase)
package com.example.project;

// ==================== 2. COMMENTS AND DOCUMENTATION ====================
/**
 * Calculates the sum of two integers.
 * 
 * @param a the first number
 * @param b the second number
 * @return the sum of a and b
 * @throws IllegalArgumentException if the numbers are negative
 */
public int sum(int a, int b) {
    if (a < 0 || b < 0) {
        throw new IllegalArgumentException("Non-negative numbers required");
    }
    return a + b;
}

// Single line comment for brief explanations

/*
 * Multi-line comment
 * for longer explanations
 */

// ==================== 3. ERROR HANDLING ====================
// ❌ Do not ignore exceptions
try {
    // operation
} catch (Exception e) {
    // DO NOT do nothing - very bad practice!
}

// ✅ At least log the error
try {
    // operation
} catch (Exception e) {
    System.err.println("Error: " + e.getMessage());
    e.printStackTrace();
}

// ==================== 4. USING try-with-resources ====================
// ✅ For AutoCloseable resources
try (BufferedReader br = new BufferedReader(new FileReader("file.txt"))) {
    String line = br.readLine();
} catch (IOException e) {
    e.printStackTrace();
}
// br is automatically closed

// ==================== 5. STRING COMPARISON ====================
// ❌ Do not use ==
if (str1 == str2) { }  // compares references, not content

// ✅ Use equals
if (str1.equals(str2)) { }
if (str1.equalsIgnoreCase(str2)) { }  // case-insensitive

// ==================== 6. VARIABLE DECLARATION ====================
// ✅ Declare variables as close as possible to their use
int result = 0;
for (int i = 0; i < 10; i++) {
    result += i;
}

// ==================== 7. AVOID MAGIC NUMBERS ====================
// ❌ Magic number
if (status == 404) { }

// ✅ Constant with meaningful name
public static final int HTTP_NOT_FOUND = 404;
if (status == HTTP_NOT_FOUND) { }

// ==================== 8. STRINGBUILDER FOR REPEATED CONCATENATIONS ====================
// ❌ Inefficient in loop
String result = "";
for (int i = 0; i < 1000; i++) {
    result += i;  // creates new String objects
}

// ✅ Efficient
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 1000; i++) {
    sb.append(i);
}
String result = sb.toString();
Checklist for quality Java code:
✓ Follow Java naming conventions
✓ Document public methods with JavaDoc
✓ Never ignore exceptions
✓ Use try-with-resources for AutoCloseable resources
✓ Use equals() to compare strings, not ==
✓ Avoid magic numbers (use constants)
✓ Use StringBuilder for concatenations in loops
✓ Make variables final when possible (immutability)
✓ Use meaningful names for variables and methods
