Kubernetes per Java

Guida completa all'orchestrazione di applicazioni Java su Kubernetes: deployment, service, ingress, configmap e best practices

Cos'è Kubernetes?

Kubernetes (K8s) è una piattaforma open-source per l'orchestrazione di container. Automatizza il deployment, lo scaling e la gestione di applicazioni containerizzate.
Perché Kubernetes per Java?
- Scalabilità automatica: scala le applicazioni Java in base al carico
- Self-healing: riavvia container falliti automaticamente
- Rollout e rollback: deploy zero-downtime delle applicazioni
- Service discovery: comunicazione automatica tra microservizi
- Gestione configurazioni: ConfigMap e Secrets per configurazioni esterne

Containerizzazione dell'App Java

📌 Dockerfile multi-stage per Spring Boot

# Stage 1: Build
FROM maven:3.9-eclipse-temurin-21 AS build
WORKDIR /app
COPY pom.xml .
RUN mvn dependency:go-offline -B
COPY src src
RUN mvn package -DskipTests

# Stage 2: Runtime
FROM eclipse-temurin:21-jre-alpine
WORKDIR /app
COPY --from=build /app/target/*.jar app.jar

# Ottimizzazione JVM per container
ENV JAVA_OPTS="-XX:+UseContainerSupport -XX:MaxRAMPercentage=75.0"

# Utente non-root per sicurezza
RUN addgroup -S spring && adduser -S spring -G spring
USER spring:spring

EXPOSE 8080
ENTRYPOINT ["sh", "-c", "java $JAVA_OPTS -jar app.jar"]

📌 Build e push dell'immagine

# Build dell'immagine
docker build -t myapp:1.0.0 .

# Tag per Docker Hub
docker tag myapp:1.0.0 myusername/myapp:1.0.0

# Push su registry
docker push myusername/myapp:1.0.0

# Test locale
docker run -p 8080:8080 myapp:1.0.0

Pod - L'unità base di Kubernetes

Un Pod è il più piccolo deployable unit in Kubernetes. Contiene uno o più container che condividono rete e storage.

📌 pod.yaml

apiVersion: v1
kind: Pod
metadata:
  name: java-app-pod
  labels:
    app: java-app
    version: v1
spec:
  containers:
  - name: java-app
    image: myusername/myapp:1.0.0
    ports:
    - containerPort: 8080
    env:
    - name: SPRING_PROFILES_ACTIVE
      value: "k8s"
    - name: DB_HOST
      value: "postgres-service"
    resources:
      requests:
        memory: "256Mi"
        cpu: "250m"
      limits:
        memory: "512Mi"
        cpu: "500m"
    livenessProbe:
      httpGet:
        path: /actuator/health
        port: 8080
      initialDelaySeconds: 30
      periodSeconds: 10
    readinessProbe:
      httpGet:
        path: /actuator/health
        port: 8080
      initialDelaySeconds: 20
      periodSeconds: 5
Comandi Pod:
kubectl apply -f pod.yaml - Crea il Pod
kubectl get pods - Lista Pod
kubectl logs java-app-pod - Log del Pod
kubectl exec -it java-app-pod -- sh - Shell nel container
kubectl delete pod java-app-pod - Elimina Pod

Deployment - Gestione dei Pod

Un Deployment gestisce il ciclo di vita dei Pod: scaling, rolling updates, rollback.

📌 deployment.yaml

apiVersion: apps/v1
kind: Deployment
metadata:
  name: java-app-deployment
  namespace: default
  labels:
    app: java-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: java-app
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 0
  template:
    metadata:
      labels:
        app: java-app
    spec:
      containers:
      - name: java-app
        image: myusername/myapp:1.0.0
        imagePullPolicy: Always
        ports:
        - containerPort: 8080
        env:
        - name: SPRING_PROFILES_ACTIVE
          valueFrom:
            configMapKeyRef:
              name: app-config
              key: spring.profiles.active
        - name: DB_PASSWORD
          valueFrom:
            secretKeyRef:
              name: db-secret
              key: password
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "512Mi"
            cpu: "500m"
        livenessProbe:
          httpGet:
            path: /actuator/health/liveness
            port: 8080
          initialDelaySeconds: 30
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /actuator/health/readiness
            port: 8080
          initialDelaySeconds: 20
          periodSeconds: 5

📌 Horizontal Pod Autoscaler (HPA)

apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: java-app-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: java-app-deployment
  minReplicas: 2
  maxReplicas: 10
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
  - type: Resource
    resource:
      name: memory
      target:
        type: Utilization
        averageUtilization: 80
Comandi Deployment:
kubectl apply -f deployment.yaml - Crea Deployment
kubectl rollout status deployment/java-app-deployment - Stato rollout
kubectl rollout history deployment/java-app-deployment - Storico versioni
kubectl rollout undo deployment/java-app-deployment - Rollback
kubectl scale deployment java-app-deployment --replicas=5 - Scaling manuale

Service - Esposizione dei Pod

Un Service fornisce un endpoint stabile per comunicare con i Pod, gestendo il load balancing.

📌 service.yaml (ClusterIP)

apiVersion: v1
kind: Service
metadata:
  name: java-app-service
spec:
  selector:
    app: java-app
  ports:
  - port: 8080
    targetPort: 8080
    protocol: TCP
  type: ClusterIP

📌 service-nodeport.yaml

apiVersion: v1
kind: Service
metadata:
  name: java-app-nodeport
spec:
  selector:
    app: java-app
  ports:
  - port: 8080
    targetPort: 8080
    nodePort: 30080
    protocol: TCP
  type: NodePort

📌 service-loadbalancer.yaml

apiVersion: v1
kind: Service
metadata:
  name: java-app-lb
spec:
  selector:
    app: java-app
  ports:
  - port: 8080
    targetPort: 8080
  type: LoadBalancer
Comandi Service:
kubectl get services - Lista Service
kubectl describe service java-app-service - Dettagli Service
kubectl port-forward service/java-app-service 8080:8080 - Port forwarding locale

📌 Utilizzo in Spring Boot

@RestController
@RequestMapping("/api/config")
@Slf4j
public class ConfigController {
    
    @Value("${app.feature.enabled:false}")
    private boolean featureEnabled;
    
    @Value("${spring.profiles.active:default}")
    private String profile;
    
    @Value("${DB_PASSWORD:not-set}")
    private String dbPassword;
    
    @GetMapping("/info")
    public Map<String, String> getConfig() {
        return Map.of(
            "profile", profile,
            "featureEnabled", String.valueOf(featureEnabled),
            "dbPassword", "****"  // Non esporre mai le password!
        );
    }
}

Ingress - Routing HTTP/HTTPS

Ingress espone servizi HTTP/HTTPS all'esterno del cluster, con routing basato su host e path.

📌 ingress.yaml

apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: java-app-ingress
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
    nginx.ingress.kubernetes.io/ssl-redirect: "true"
    cert-manager.io/cluster-issuer: "letsencrypt-prod"
spec:
  ingressClassName: nginx
  tls:
  - hosts:
    - api.myapp.com
    secretName: tls-secret
  rules:
  - host: api.myapp.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: java-app-service
            port:
              number: 8080
      - path: /api/v2
        pathType: Prefix
        backend:
          service:
            name: java-app-v2-service
            port:
              number: 8080

Persistent Volume e Persistent Volume Claim

📌 persistent-volume.yaml

apiVersion: v1
kind: PersistentVolume
metadata:
  name: postgres-pv
spec:
  capacity:
    storage: 10Gi
  accessModes:
    - ReadWriteOnce
  hostPath:
    path: /data/postgres
---
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: postgres-pvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 5Gi

📌 Utilizzo in Deployment

apiVersion: apps/v1
kind: Deployment
metadata:
  name: postgres
spec:
  replicas: 1
  selector:
    matchLabels:
      app: postgres
  template:
    metadata:
      labels:
        app: postgres
    spec:
      containers:
      - name: postgres
        image: postgres:15
        env:
        - name: POSTGRES_PASSWORD
          valueFrom:
            secretKeyRef:
              name: db-secret
              key: password
        volumeMounts:
        - name: postgres-storage
          mountPath: /var/lib/postgresql/data
      volumes:
      - name: postgres-storage
        persistentVolumeClaim:
          claimName: postgres-pvc

Helm - Package Manager per Kubernetes

Helm è il package manager di Kubernetes. Permette di definire, installare e aggiornare applicazioni complesse.

📌 Struttura di un Chart Helm

myapp-chart/
├── Chart.yaml          # Metadata del chart
├── values.yaml         # Valori di default
├── templates/          # Template Kubernetes
│   ├── deployment.yaml
│   ├── service.yaml
│   ├── configmap.yaml
│   └── ingress.yaml
└── charts/             # Dipendenze

📌 values.yaml

replicaCount: 3
image:
  repository: myusername/myapp
  tag: "1.0.0"
  pullPolicy: Always

service:
  type: ClusterIP
  port: 8080

ingress:
  enabled: true
  host: api.myapp.com

resources:
  requests:
    memory: "256Mi"
    cpu: "250m"
  limits:
    memory: "512Mi"
    cpu: "500m"

config:
  springProfilesActive: "k8s"
  featureEnabled: true

📌 template/deployment.yaml (con Go template)

apiVersion: apps/v1
kind: Deployment
metadata:
  name: {{ .Values.appName }}-deployment
spec:
  replicas: {{ .Values.replicaCount }}
  selector:
    matchLabels:
      app: {{ .Values.appName }}
  template:
    metadata:
      labels:
        app: {{ .Values.appName }}
    spec:
      containers:
      - name: {{ .Values.appName }}
        image: "{{ .Values.image.repository }}:{{ .Values.image.tag }}"
        ports:
        - containerPort: {{ .Values.service.port }}
        env:
        - name: SPRING_PROFILES_ACTIVE
          value: {{ .Values.config.springProfilesActive }}
        resources:
          requests:
            memory: {{ .Values.resources.requests.memory }}
            cpu: {{ .Values.resources.requests.cpu }}
          limits:
            memory: {{ .Values.resources.limits.memory }}
            cpu: {{ .Values.resources.limits.cpu }}
Comandi Helm:
helm create myapp-chart - Crea nuovo chart
helm install myapp ./myapp-chart - Installa chart
helm upgrade myapp ./myapp-chart - Aggiorna
helm rollback myapp 1 - Rollback alla revisione 1
helm uninstall myapp - Disinstalla
helm repo add bitnami https://charts.bitnami.com/bitnami - Aggiunge repo

Best Practices per Java su Kubernetes

Regole fondamentali per applicazioni Java su K8s
// ==================== 1. JVM PER CONTAINER ====================
// Configura la JVM per rispettare i limiti del container
// -XX:+UseContainerSupport (attivo di default in Java 10+)
// -XX:MaxRAMPercentage=75.0 (usa il 75% della memoria del container)

// In Dockerfile
ENV JAVA_OPTS="-XX:+UseContainerSupport -XX:MaxRAMPercentage=75.0"

// ==================== 2. GESTIONE DELLA MEMORIA ====================
// In deployment.yaml
resources:
  requests:
    memory: "512Mi"   # Minimo garantito
    cpu: "250m"
  limits:
    memory: "1024Mi"  # Massimo utilizzabile
    cpu: "1000m"

// ==================== 3. LIVENESS E READINESS PROBE ====================
// Spring Boot Actuator endpoints
livenessProbe:
  httpGet:
    path: /actuator/health/liveness
    port: 8080
  initialDelaySeconds: 60
  periodSeconds: 10

readinessProbe:
  httpGet:
    path: /actuator/health/readiness
    port: 8080
  initialDelaySeconds: 30
  periodSeconds: 5

// ==================== 4. GRACEFUL SHUTDOWN ====================
// Spring Boot 2.3+ supporta graceful shutdown
# application.yml
server:
  shutdown: graceful

spring:
  lifecycle:
    timeout-per-shutdown-phase: 30s

// ==================== 5. DISTRIBUTED TRACING ====================
// Aggiungi correlation ID per tracing distribuito
@Component
public class CorrelationIdFilter implements Filter {
    @Override
    public void doFilter(ServletRequest request, ServletResponse response, 
                         FilterChain chain) throws IOException {
        String correlationId = UUID.randomUUID().toString();
        MDC.put("correlationId", correlationId);
        chain.doFilter(request, response);
        MDC.clear();
    }
}

// ==================== 6. LOGGING STRUTTURATO ====================
// Usa JSON logging per integrazione con ELK/Loki
# application.yml
logging:
  pattern:
    console: "{\"timestamp\":\"%d{ISO8601}\",\"level\":\"%p\",\"thread\":\"%t\",\"logger\":\"%c\",\"message\":\"%m\",\"correlationId\":\"%X{correlationId}\"}"
  level:
    com.myapp: DEBUG
Checklist per Java su Kubernetes:
✓ Usa immagini base leggere (Alpine, Distroless)
✓ Configura JVM con -XX:+UseContainerSupport
✓ Imposta limiti di memoria e CPU
✓ Configura liveness e readiness probe
✓ Usa ConfigMap e Secret per configurazioni
✓ Implementa graceful shutdown
✓ Usa structured logging (JSON)
✓ Aggiungi correlation ID per tracing
✓ Usa Helm per gestire configurazioni
✓ Monitora con Prometheus e Grafana
