Gestione delle Eccezioni in Java

try-catch-finally, throw, throws, eccezioni custom, try-with-resources e best practices

Cos'è un'eccezione?

Un'eccezione in Java è un evento che interrompe il normale flusso di esecuzione di un programma. Java fornisce un robusto meccanismo per gestire gli errori attraverso le eccezioni, permettendo di separare la logica di business dalla gestione degli errori.
Vantaggi della gestione delle eccezioni:
- Separazione della logica di errore dalla logica di business
- Propagazione degli errori lungo la call stack
- Raggruppamento di tipi di errore
- Gestione centralizzata degli errori

Gerarchia delle Eccezioni

Throwable è la classe base di tutte le eccezioni
// Gerarchia delle eccezioni in Java:
//
//                    Throwable
//                   /        \
//              Error        Exception
//              /    \       /        \
//   OutOfMemoryError  StackOverflowError  RuntimeException  IOException
//                                         /        \
//                              NullPointerException  IllegalArgumentException
//
// - Error: errori gravi dell'ambiente (non recuperabili)
// - Exception (checked): obbligano a gestione (try-catch o throws)
// - RuntimeException (unchecked): non obbligano a gestione

public class GerarchiaEccezioni {
    public static void main(String[] args) {
        // RuntimeException (non obbliga try-catch)
        String s = null;
        try {
            s.length();  // NullPointerException
        } catch (NullPointerException e) {
            System.out.println("Gestita NPE: " + e.getMessage());
        }
        
        // Checked Exception (obbliga try-catch o throws)
        try {
            FileReader file = new FileReader("file.txt");
        } catch (FileNotFoundException e) {
            System.out.println("File non trovato!");
        }
    }
}
Nota: Le eccezioni checked (come IOException) DEVONO essere gestite con try-catch o dichiarate con throws. Le unchecked (RuntimeException) sono opzionali ma consigliate.

try-catch-finally

Blocchi fondamentali per la gestione delle eccezioni
public class TryCatchFinallyDemo {
    public static void main(String[] args) {
        System.out.println(divisioneSicura(10, 2));
        System.out.println(divisioneSicura(10, 0));
    }
    
    public static int divisioneSicura(int a, int b) {
        try {
            System.out.println("Tentativo divisione: " + a + "/" + b);
            return a / b;
        } catch (ArithmeticException e) {
            System.err.println("ERRORE: " + e.getMessage());
            return -1;
        } finally {
            System.out.println("Finally - sempre eseguito");
        }
    }
}
Output:
Tentativo divisione: 10/2
Finally - sempre eseguito
5
Tentativo divisione: 10/0
ERRORE: / by zero
Finally - sempre eseguito
-1
Importante: Il blocco finally viene eseguito SEMPRE, anche se c'è un return nel try o nel catch. È ideale per chiudere risorse.

throw e throws

throw → lancia un'eccezione | throws → dichiara che un metodo può lanciare eccezioni
public class ThrowThrowsDemo {
    
    // throws dichiara che il metodo può lanciare un'eccezione
    public static void verificaEta(int eta) throws IllegalArgumentException {
        if (eta < 0 || eta > 150) {
            // throw lancia effettivamente l'eccezione
            throw new IllegalArgumentException("Età non valida: " + eta);
        }
        System.out.println("Età valida: " + eta);
    }
    
    // Multiplo throws
    public static void leggiFile(String path) throws FileNotFoundException, IOException {
        FileReader reader = new FileReader(path);
        int data = reader.read();
        reader.close();
    }
    
    // Rilancio eccezione con causa
    public static void elaboraDati() throws BusinessException {
        try {
            // operazione che può fallire
            leggiFile("dati.txt");
        } catch (IOException e) {
            // Preserva la causa originale
            throw new BusinessException("Errore elaborazione dati", e);
        }
    }
    
    public static void main(String[] args) {
        try {
            verificaEta(200);  // Lancia eccezione
        } catch (IllegalArgumentException e) {
            System.out.println("Catturata: " + e.getMessage());
        }
        
        try {
            verificaEta(25);   // Ok
        } catch (IllegalArgumentException e) {
            System.out.println(e.getMessage());
        }
        
        // Gestione di più eccezioni in un unico catch (Java 7+)
        try {
            leggiFile("test.txt");
        } catch (FileNotFoundException | IOException e) {
            System.out.println("Errore I/O: " + e.getMessage());
        }
    }
}

class BusinessException extends Exception {
    public BusinessException(String message, Throwable cause) {
        super(message, cause);
    }
}
Output:
Catturata: Età non valida: 200
Età valida: 25

Eccezioni Custom

Creare le proprie eccezioni per errori specifici del dominio
// ==================== ECCEZIONE UNCHECKED ====================
// Estende RuntimeException (non obbliga try-catch)
class SaldoInsufficienteException extends RuntimeException {
    private final double saldoCorrente;
    private final double importoRichiesto;
    
    public SaldoInsufficienteException(double saldoCorrente, double importoRichiesto) {
        super(String.format("Saldo insufficiente: €%.2f disponibili, €%.2f richiesti", 
              saldoCorrente, importoRichiesto));
        this.saldoCorrente = saldoCorrente;
        this.importoRichiesto = importoRichiesto;
    }
    
    public double getSaldoCorrente() { return saldoCorrente; }
    public double getImportoRichiesto() { return importoRichiesto; }
}

// ==================== ECCEZIONE CHECKED ====================
// Estende Exception (obbliga try-catch o throws)
class ContoNonTrovatoException extends Exception {
    private final String iban;
    private final LocalDateTime timestamp;
    
    public ContoNonTrovatoException(String iban) {
        super("Conto con IBAN " + iban + " non trovato nel sistema");
        this.iban = iban;
        this.timestamp = LocalDateTime.now();
    }
    
    public ContoNonTrovatoException(String iban, Throwable cause) {
        super("Conto con IBAN " + iban + " non trovato nel sistema", cause);
        this.iban = iban;
        this.timestamp = LocalDateTime.now();
    }
    
    public String getIban() { return iban; }
    public LocalDateTime getTimestamp() { return timestamp; }
}

// ==================== CLASSE CONTO BANCARIO ====================
class ContoBancario {
    private double saldo;
    private String iban;
    private String proprietario;
    
    public ContoBancario(String iban, String proprietario, double saldoIniziale) {
        this.iban = iban;
        this.proprietario = proprietario;
        this.saldo = saldoIniziale;
    }
    
    public void preleva(double importo) {
        if (importo > saldo) {
            throw new SaldoInsufficienteException(saldo, importo);
        }
        saldo -= importo;
        System.out.printf("Prelievo di €%.2f effettuato. Nuovo saldo: €%.2f%n", importo, saldo);
    }
    
    public void deposita(double importo) {
        if (importo <= 0) {
            throw new IllegalArgumentException("Importo deposito non valido: " + importo);
        }
        saldo += importo;
        System.out.printf("Deposito di €%.2f effettuato. Nuovo saldo: €%.2f%n", importo, saldo);
    }
    
    public static ContoBancario trovaPerIban(String iban) throws ContoNonTrovatoException {
        // Simulazione ricerca
        if (!iban.equals("IT1234567890")) {
            throw new ContoNonTrovatoException(iban);
        }
        return new ContoBancario(iban, "Mario Rossi", 1000.0);
    }
    
    public double getSaldo() { return saldo; }
    public String getIban() { return iban; }
    public String getProprietario() { return proprietario; }
}

public class CustomExceptionDemo {
    public static void main(String[] args) {

        try {

try-with-resources (AutoCloseable)

Java 7+ - Chiusura automatica delle risorse

📌 VECCHIO MODO (Java 6 e precedenti)

// Senza try-with-resources - tanto boilerplate!
BufferedReader br = null;
try {
    br = new BufferedReader(new FileReader("file.txt"));
    String linea = br.readLine();
    System.out.println(linea);
} catch (IOException e) {
    e.printStackTrace();
} finally {
    if (br != null) {
        try {
            br.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

📌 NUOVO MODO (try-with-resources)

// Con try-with-resources - molto più pulito!
try (BufferedReader br = new BufferedReader(new FileReader("file.txt"))) {
    String linea = br.readLine();
    System.out.println(linea);
} catch (IOException e) {
    System.err.println("Errore lettura: " + e.getMessage());
}
// br viene chiuso automaticamente!

📌 MULTIPLE RESOURCES

// Gestione di più risorse contemporaneamente
try (FileInputStream fis = new FileInputStream("input.dat");
     FileOutputStream fos = new FileOutputStream("output.dat");
     BufferedReader reader = new BufferedReader(new FileReader("log.txt"))) {
    
    int data;
    while ((data = fis.read()) != -1) {
        fos.write(data);
    }
    System.out.println("Copia completata");
    
} catch (IOException e) {
    e.printStackTrace();
}
// Tutte e tre le risorse vengono chiuse automaticamente!

📌 CLASSE CUSTOM AUTO-CLOSEABLE

class CustomResource implements AutoCloseable {
    private String name;
    
    public CustomResource(String name) {
        this.name = name;
        System.out.println("🔓 Apertura risorsa: " + name);
    }
    
    public void doSomething() {
        System.out.println("⚙️ Utilizzo risorsa: " + name);
    }
    
    @Override
    public void close() throws Exception {
        System.out.println("🔒 Chiusura risorsa: " + name);
    }
}

// Utilizzo
try (CustomResource res1 = new CustomResource("DB Connection");
     CustomResource res2 = new CustomResource("File Handler")) {
    
    res1.doSomething();
    res2.doSomething();
    
} catch (Exception e) {
    e.printStackTrace();
}
// Output:
// 🔓 Apertura risorsa: DB Connection
// 🔓 Apertura risorsa: File Handler
// ⚙️ Utilizzo risorsa: DB Connection
// ⚙️ Utilizzo risorsa: File Handler
// 🔒 Chiusura risorsa: File Handler
// 🔒 Chiusura risorsa: DB Connection
Vantaggi di try-with-resources:
- Codice più pulito e leggibile
- Chiusura automatica anche in caso di eccezioni
- Supporto per risorse multiple
- Le risorse vengono chiuse in ordine inverso di apertura

Best Practices per le Eccezioni

Regole fondamentali per scrivere codice robusto
// ==================== 1. NON IGNORARE MAI LE ECCEZIONI ====================
// ❌ SBAGLIATO
try {
    // codice
} catch (Exception e) {
    // Non fare nulla - pessima pratica!
}

// ✅ CORRETTO - almeno logga l'errore
try {
    // codice
} catch (Exception e) {
    logger.error("Errore durante l'operazione", e);
    // o e.printStackTrace();
}

// ==================== 2. USA ECCEZIONI SPECIFICHE ====================
// ❌ SBAGLIATO - troppo generico
try {
    // codice
} catch (Exception e) {
    // cattura tutto
}

// ✅ CORRETTO - eccezioni specifiche
try {
    // codice
} catch (FileNotFoundException e) {
    System.out.println("File non trovato");
} catch (IOException e) {
    System.out.println("Errore I/O");
} catch (SQLException e) {
    System.out.println("Errore database");
}

// ==================== 3. PROPAGA LE ECCEZIONI QUANDO APPROPRIATO ====================
// ✅ Quando il metodo chiamante deve sapere dell'errore
public void leggiConfigurazione() throws IOException {
    // se fallisce, il chiamante decide come gestire
}

// ==================== 4. USA FINALLY PER PULIRE RISORSE (O TRY-WITH-RESOURCES) ====================
Connection conn = null;
try {
    conn = dataSource.getConnection();
    // usa connection
} finally {
    if (conn != null) {
        try {
            conn.close();
        } catch (SQLException e) {
            // logga ma non rilanciare
        }
    }
}

// ==================== 5. NON USARE ECCEZIONI PER IL FLUSSO NORMALE ====================
// ❌ SBAGLIATO
try {
    return array[index];
} catch (IndexOutOfBoundsException e) {
    return null;  // non usare eccezioni per controlli logici!
}

// ✅ CORRETTO
if (index >= 0 && index < array.length) {
    return array[index];
} else {
    return null;
}

// ==================== 6. FORNISCI MESSAGGI SIGNIFICATIVI ====================
// ❌ SBAGLIATO
throw new RuntimeException("Errore");

// ✅ CORRETTO
throw new IllegalArgumentException(
    String.format("Valore %d non valido. Deve essere compreso tra %d e %d", 
                  valore, min, max)
);

// ==================== 7. PRESERVA LA STACK TRACE ORIGINALE ====================
// ❌ SBAGLIATO - perde la causa originale
try {
    // operazione
} catch (SQLException e) {
    throw new MyBusinessException("Errore DB");
}

// ✅ CORRETTO - preserva la causa
try {
    // operazione
} catch (SQLException e) {
    throw new MyBusinessException("Errore DB", e);  // passa la causa
}

// ==================== 8. DOCUMENTA LE ECCEZIONI ====================
/**
 * Calcola la radice quadrata di un numero.
 * 
 * @param numero il numero di cui calcolare la radice quadrata
 * @return la radice quadrata del numero
 * @throws IllegalArgumentException se il numero è negativo
 */
public double radiceQuadrata(double numero) throws IllegalArgumentException {
    if (numero < 0) {
        throw new IllegalArgumentException("Numero negativo: " + numero);
    }
    return Math.sqrt(numero);
}
Checklist per eccezioni di qualità:
✓ Usa eccezioni specifiche (non Exception generica)
✓ Fornisci messaggi di errore informativi
✓ Non nascondere mai le eccezioni (catch vuoto)
✓ Usa try-with-resources per risorse AutoCloseable
✓ Documenta le eccezioni con @throws nei JavaDoc
✓ Preserva la stack trace originale (causa)

📌 ESEMPIO 1: Validazione input utente

import java.util.Scanner;

public class InputValidator {
    
    public static int parseIntSicuro(String input) {
        try {
            return Integer.parseInt(input);
        } catch (NumberFormatException e) {
            System.err.println("Input non valido: '" + input + "' non è un numero");
            return 0;  // valore di default
        }
    }
    
    public static Optional<Integer> parseIntOptional(String input) {
        try {
            return Optional.of(Integer.parseInt(input));
        } catch (NumberFormatException e) {
            return Optional.empty();
        }
    }
    
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        
        System.out.print("Inserisci un numero: ");
        String input = scanner.nextLine();
        
        // Modo 1: con valore di default
        int numero1 = parseIntSicuro(input);
        System.out.println("Numero (default): " + numero1);
        
        // Modo 2: con Optional
        Optional<Integer> numero2 = parseIntOptional(input);
        numero2.ifPresentOrElse(
            n -> System.out.println("Numero (optional): " + n),
            () -> System.out.println("Input non valido, nessun numero")
        );
        
        scanner.close();
    }
}

📌 ESEMPIO 2: Lettura file con multiple eccezioni

import java.io.*;
import java.nio.file.*;
import java.util.*;

public class FileProcessor {
    
    public static List<String> leggiRighe(String path) {
        List<String> righe = new ArrayList<>();
        
        try (BufferedReader reader = new BufferedReader(new FileReader(path))) {
            String linea;
            while ((linea = reader.readLine()) != null) {
                righe.add(linea);
            }
        } catch (FileNotFoundException e) {
            System.err.println("File non trovato: " + path);
            // Crea il file di default
            creaFileDefault(path);
            // Prova a leggerlo di nuovo
            return leggiRighe(path);
        } catch (IOException e) {
            System.err.println("Errore durante la lettura: " + e.getMessage());
        }
        
        return righe;
    }
    
    private static void creaFileDefault(String path) {
        try (FileWriter writer = new FileWriter(path)) {
            writer.write("# File creato automaticamente\n");
            writer.write("riga1\n");
            writer.write("riga2\n");
            writer.write("riga3\n");
            System.out.println("File default creato: " + path);
        } catch (IOException e) {
            System.err.println("Impossibile creare il file default: " + e.getMessage());
        }
    }
    
    public static void main(String[] args) {
        List<String> righe = leggiRighe("config.txt");
        System.out.println("Righe lette: " + righe.size());
        righe.forEach(System.out::println);
    }
}

📌 ESEMPIO 3: Gestione errori in API REST (simulazione)

import java.time.LocalDateTime;
import java.util.*;

// ==================== MODELLI DI RISPOSTA ====================
class ErrorResponse {
    private String code;
    private String message;
    private LocalDateTime timestamp;
    private Map<String, String> details;
    
    public ErrorResponse(String code, String message) {
        this.code = code;
        this.message = message;
        this.timestamp = LocalDateTime.now();
        this.details = new HashMap<>();
    }
    
    public ErrorResponse addDetail(String key, String value) {
        details.put(key, value);
        return this;
    }
    
    // Getters
    public String getCode() { return code; }
    public String getMessage() { return message; }
    public LocalDateTime getTimestamp() { return timestamp; }
    public Map<String, String> getDetails() { return details; }
    
    @Override
    public String toString() {
        return String.format("ErrorResponse{code='%s', message='%s', timestamp=%s, details=%s}",
            code, message, timestamp, details);
    }
}

// ==================== ECCEZIONI PERSONALIZZATE ====================
class ResourceNotFoundException extends RuntimeException {
    private final String resourceType;
    private final String resourceId;
    
    public ResourceNotFoundException(String resourceType, String resourceId) {
        super(String.format("%s con ID '%s' non trovato", resourceType, resourceId));
        this.resourceType = resourceType;
        this.resourceId = resourceId;
    }
    
    public String getResourceType() { return resourceType; }
    public String getResourceId() { return resourceId; }
}

class ValidationException extends RuntimeException {
    private final Map<String, String> errors;
    
    public ValidationException(Map<String, String> errors) {
        super("Errore di validazione");
        this.errors = errors;
    }
    
    public Map<String, String> getErrors() { return errors; }
}

// ==================== GESTORE CENTRALIZZATO ECCEZIONI ====================
class GlobalExceptionHandler {
    
    public ErrorResponse handleNotFound(ResourceNotFoundException e) {
        return new ErrorResponse("NOT_FOUND", e.getMessage())
            .addDetail("resourceType", e.getResourceType())
            .addDetail("resourceId", e.getResourceId());
    }
    
    public ErrorResponse handleValidation(ValidationException e) {
        ErrorResponse response = new ErrorResponse("VALIDATION_ERROR", e.getMessage());
        e.getErrors().forEach(response::addDetail);
        return response;
    }
    
    public ErrorResponse handleGeneric(Exception e) {
        return new ErrorResponse("INTERNAL_ERROR", "Errore interno del server")
            .addDetail("exception", e.getClass().getSimpleName());
    }
}

// ==================== ESEMPIO DI UTILIZZO ====================
public class ApiExceptionDemo {
    
    private static final GlobalExceptionHandler exceptionHandler = new GlobalExceptionHandler();
    
    public static void main(String[] args) {
        // Simula una chiamata API
        System.out.println("=== Test ResourceNotFoundException ===\n");
        try {
            trovaUtente("999");
        } catch (ResourceNotFoundException e) {
            ErrorResponse response = exceptionHandler.handleNotFound(e);
            System.out.println(response);
        }
        
        System.out.println("\n=== Test ValidationException ===\n");
        try {
            creaUtente(null, "email");
        } catch (ValidationException e) {
            ErrorResponse response = exceptionHandler.handleValidation(e);
            System.out.println(response);
        }
        
        System.out.println("\n=== Test generico ===\n");
        try {
            operazioneRischiosa();
        } catch (Exception e) {
            ErrorResponse response = exceptionHandler.handleGeneric(e);
            System.out.println(response);
        }
    }
    
    static Utente trovaUtente(String id) {
        if (!id.equals("123")) {
            throw new ResourceNotFoundException("Utente", id);
        }
        return new Utente(id, "Mario Rossi", "mario@email.com");
    }
    
    static void creaUtente(String nome, String email) {
        Map<String, String> errors = new HashMap<>();
        
        if (nome == null || nome.isEmpty()) {
            errors.put("nome", "Il nome è obbligatorio");
        }
        if (email == null || !email.contains("@")) {
            errors.put("email", "Email non valida");
        }
        
        if (!errors.isEmpty()) {
            throw new ValidationException(errors);
        }
        
        System.out.println("Utente creato: " + nome);
    }
    
    static void operazioneRischiosa() {
        throw new RuntimeException("Errore imprevisto nel database");
    }
    
    static class Utente {
        String id, nome, email;
        Utente(String id, String nome, String email) {
            this.id = id; this.nome = nome; this.email = email;
        }
    }
}
