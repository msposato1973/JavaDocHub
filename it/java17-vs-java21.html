Pattern Matching per switch

Java 17 → Switch expression senza pattern
Java 21 → Pattern matching su switch (stabile)
Java 17
// Java 17 - necessita instanceof prima dello switch
String formatta(Object obj) {
    if (obj instanceof Integer i) {
        return "Intero: " + i;
    } else if (obj instanceof String s) {
        return "Stringa: " + s;
    } else if (obj == null) {
        return "Null";
    }
    return "Sconosciuto";
}
Java 21 STABILE
// Java 21 - pattern matching direttamente nello switch
String formatta(Object obj) {
    return switch (obj) {
        case Integer i -> "Intero: " + i;
        case String s -> "Stringa: " + s;
        case null -> "Null";
        default -> "Sconosciuto";
    };
}

// Con guardie (when clause)
String descrivi(Object obj) {
    return switch (obj) {
        case Integer i when i > 0 -> "Positivo";
        case Integer i when i < 0 -> "Negativo";
        case Integer i -> "Zero";
        case String s when s.length() > 10 -> "Stringa lunga";
        case String s -> "Stringa corta";
        default -> "Altro";
    };
}
Java 21 vantaggio: Codice più pulito e leggibile, pattern matching integrato nello switch.

Record Patterns

Java 17 → Record senza destrutturazione
Java 21 → Record patterns (stabile)
Java 17
record Punto(int x, int y) {}
record Rettangolo(Punto altoSinistra, Punto bassoDestra) {}

// Decomposizione manuale
void stampa(Rettangolo r) {
    if (r instanceof Rettangolo) {
        Punto p1 = r.altoSinistra();
        Punto p2 = r.bassoDestra();
        System.out.println(p1.x() + "," + p1.y());
    }
}
Java 21 STABILE
record Punto(int x, int y) {}
record Rettangolo(Punto altoSinistra, Punto bassoDestra) {}

// Destrutturazione automatica
void stampa(Rettangolo r) {
    if (r instanceof Rettangolo(Punto(int x1, int y1), 
                                Punto(int x2, int y2))) {
        System.out.println(x1 + "," + y1 + " - " + x2 + "," + y2);
    }
}

// In switch
int area(Object obj) {
    return switch (obj) {
        case Rettangolo(Punto(int x1, int y1), 
                       Punto(int x2, int y2)) -> (x2 - x1) * (y2 - y1);
        default -> 0;
    };
}
Java 21 vantaggio: Destrutturazione automatica dei Record, codice più espressivo.

Sequenced Collections

Java 17 → Interfacce separate per liste ordinate
Java 21 → Nuova interfaccia SequencedCollection
import java.util.*;

// Java 21 - Nuove interfacce per collezioni ordinate
// SequencedCollection, SequencedSet, SequencedMap

public class SequencedCollectionsDemo {
    public static void main(String[] args) {
        
        // ========== SEQUENCED COLLECTION ==========
        SequencedCollection<String> list = new ArrayList<>(List.of("A", "B", "C"));
        
        // Nuovi metodi
        String first = list.getFirst();     // "A"
        String last = list.getLast();       // "C"
        
        list.addFirst("Z");    // [Z, A, B, C]
        list.addLast("D");     // [Z, A, B, C, D]
        
        list.removeFirst();    // remove "Z"
        list.removeLast();     // remove "D"
        
        // Reverse order view
        SequencedCollection<String> reversed = list.reversed();
        System.out.println("Reversed: " + reversed);  // [C, B, A]
        
        // ========== SEQUENCED MAP ==========
        SequencedMap<String, Integer> map = new LinkedHashMap<>();
        map.put("A", 1);
        map.put("B", 2);
        map.put("C", 3);
        
        // Primo e ultimo entry
        Map.Entry<String, Integer> firstEntry = map.firstEntry();    // A=1
        Map.Entry<String, Integer> lastEntry = map.lastEntry();      // C=3
        
        // Reversed map
        SequencedMap<String, Integer> reversedMap = map.reversed();
        
        // ========== VANTAGGI ==========
        // Prima (Java 17) per ottenere primo e ultimo elemento:
        // List: list.get(0), list.get(list.size()-1)
        // Ora: list.getFirst(), list.getLast()
    }
}
Output:
Reversed: [C, B, A]
First entry: A=1
Last entry: C=3

Virtual Threads (Project Loom)

Java 17 → Platform Threads (pesanti)
Java 21 → Virtual Threads (leggeri) STABILE
import java.util.concurrent.*;
import java.util.stream.IntStream;

public class VirtualThreadsDemo {
    public static void main(String[] args) throws Exception {
        
        // ========== PIATFORM THREADS (Java 17) ==========
        System.out.println("=== Platform Threads (pesanti) ===");
        long startPlatform = System.currentTimeMillis();
        
        try (ExecutorService executor = Executors.newFixedThreadPool(100)) {
            IntStream.range(0, 1000).forEach(i -> {
                executor.submit(() -> {
                    try { Thread.sleep(10); } catch (InterruptedException e) {}
                });
            });
            executor.shutdown();
            executor.awaitTermination(1, TimeUnit.MINUTES);
        }
        
        long endPlatform = System.currentTimeMillis();
        System.out.println("Platform threads: " + (endPlatform - startPlatform) + "ms");
        
        // ========== VIRTUAL THREADS (Java 21) ==========
        System.out.println("\n=== Virtual Threads (leggeri) ===");
        long startVirtual = System.currentTimeMillis();
        
        try (ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor()) {
            IntStream.range(0, 1_000_000).forEach(i -> {
                executor.submit(() -> {
                    try { Thread.sleep(10); } catch (InterruptedException e) {}
                });
            });
            executor.shutdown();
            executor.awaitTermination(1, TimeUnit.MINUTES);
        }
        
        long endVirtual = System.currentTimeMillis();
        System.out.println("Virtual threads: " + (endVirtual - startVirtual) + "ms");
        
        // ========== CREAZIONE VIRTUAL THREADS ==========
        System.out.println("\n=== Creazione Virtual Threads ===");
        
        // Metodo 1: Thread.ofVirtual()
        Thread vt1 = Thread.ofVirtual()
            .name("virtual-thread-1")
            .start(() -> System.out.println("Ciao da virtual thread"));
        
        // Metodo 2: Thread.startVirtualThread()
        Thread vt2 = Thread.startVirtualThread(() -> 
            System.out.println("Ciao da altro virtual thread"));
        
        // Metodo 3: Executors.newVirtualThreadPerTaskExecutor()
        try (ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor()) {
            executor.submit(() -> System.out.println("Task su virtual thread"));
        }
        
        vt1.join();
        vt2.join();
        
        // ========== CONFRONTO PRESTAZIONI ==========
        System.out.println("\n=== Confronto prestazioni ===");
        System.out.println("Platform threads: ~1000 thread max");
        System.out.println("Virtual threads: milioni di thread simultanei!");
    }
}
Output:
Platform threads: ~250ms (con 1000 thread)
Virtual threads: ~500ms (con 1.000.000 thread!)
I virtual thread sono estremamente leggeri e permettono milioni di thread simultanei.

Structured Concurrency (Preview)

Java 17 → Gestione concorrenza manuale
Java 21 → Structured Concurrency (Preview)
import java.util.concurrent.*;
import java.util.concurrent.StructuredTaskScope.*;

// Java 21 - StructuredTaskScope (Preview)
// Richiede: --enable-preview

public class StructuredConcurrencyDemo {
    
    // ========== SENZA STRUCTURED CONCURRENCY ==========
    static class UserProfile {
        String userId, name, orders, reviews;
    }
    
    UserProfile fetchProfileLegacy(String userId) throws Exception {
        // Con CompletableFuture (non strutturato)
        CompletableFuture<String> nameFuture = CompletableFuture.supplyAsync(() -> 
            fetchName(userId));
        CompletableFuture<String> ordersFuture = CompletableFuture.supplyAsync(() -> 
            fetchOrders(userId));
        CompletableFuture<String> reviewsFuture = CompletableFuture.supplyAsync(() -> 
            fetchReviews(userId));
        
        // Attende tutti, ma se uno fallisce, gli altri continuano
        CompletableFuture.allOf(nameFuture, ordersFuture, reviewsFuture).join();
        
        UserProfile profile = new UserProfile();
        profile.name = nameFuture.get();
        profile.orders = ordersFuture.get();
        profile.reviews = reviewsFuture.get();
        return profile;
    }
    
    // ========== CON STRUCTURED CONCURRENCY (Java 21) ==========
    UserProfile fetchProfileStructured(String userId) throws Exception {
        // Scope strutturato: tutti i subtask vengono gestiti insieme
        try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
            
            Subtask<String> nameTask = scope.fork(() -> fetchName(userId));
            Subtask<String> ordersTask = scope.fork(() -> fetchOrders(userId));
            Subtask<String> reviewsTask = scope.fork(() -> fetchReviews(userId));
            
            // Se un subtask fallisce, tutti vengono cancellati
            scope.join();           // Attende tutti
            scope.throwIfFailed();  // Propaga eventuali eccezioni
            
            UserProfile profile = new UserProfile();
            profile.name = nameTask.get();
            profile.orders = ordersTask.get();
            profile.reviews = reviewsTask.get();
            return profile;
        }
        // All'uscita dal try, tutti i thread vengono automaticamente terminati
    }
    
    private String fetchName(String userId) {
        return "Mario Rossi";
    }
    
    private String fetchOrders(String userId) {
        return "5 ordini";
    }
    
    private String fetchReviews(String userId) {
        return "10 recensioni";
    }
    
    public static void main(String[] args) throws Exception {
        StructuredConcurrencyDemo demo = new StructuredConcurrencyDemo();
        UserProfile profile = demo.fetchProfileStructured("123");
        System.out.println("Profile: " + profile.name + ", " + profile.orders);
    }
}
Output:
Profile: Mario Rossi, 5 ordini

String Templates (Preview)

Java 17 → Concatenazione o String.format()
Java 21 → String Templates (Preview)
// Java 21 - String Templates (Preview)
// Richiede: --enable-preview

public class StringTemplatesDemo {
    public static void main(String[] args) {
        String nome = "Mario";
        int eta = 30;
        double prezzo = 49.99;
        
        // ========== VECCHI METODI ==========
        System.out.println("=== Vecchi metodi ===");
        
        // Concatenazione
        String msg1 = "Utente: " + nome + ", età: " + eta;
        
        // String.format()
        String msg2 = String.format("Utente: %s, età: %d", nome, eta);
        
        // ========== STRING TEMPLATES (Java 21) ==========
        System.out.println("\n=== String Templates (Java 21) ===");
        
        // Template semplice
        String msg3 = STR."Utente: \{nome}, età: \{eta}";
        System.out.println(msg3);
        
        // Espressioni nei template
        String msg4 = STR."Tra un anno \{nome} avrà \{eta + 1} anni";
        System.out.println(msg4);
        
        // Template multilinea
        String json = STR."""
            {
                "nome": "\{nome}",
                "eta": \{eta},
                "prezzo": \{prezzo}
            }
            """;
        System.out.println(json);
        
        // Template con processor personalizzato
        // FMT processor per formattazione
        String formattato = FMT."Prezzo: %.2f\{prezzo}";
        System.out.println(formattato);
        
        // ========== PROCESSOR PERSONALIZZATO ==========
        // JSON processor
        var JSON = StringTemplate.Processor.of((StringTemplate st) -> {
            // Validazione e trasformazione JSON
            return st.interpolate();
        });
        
        // SQL processor (previene SQL injection)
        var SQL = StringTemplate.Processor.of((StringTemplate st) -> {
            // Sanitizzazione automatica
            return st.interpolate();
        });
    }
}
Output:

Tra un anno Mario avrà 31 anni
{"nome": "Mario", "eta": 30, "prezzo": 49.99}

Unnamed Variables

Java 17 → Variabili con nome obbligatorio
Java 21 → Unnamed variables (_) STABILE
import java.util.*;
import java.util.function.*;

// Java 21 - Unnamed variables ( _ )

public class UnnamedVariablesDemo {
    public static void main(String[] args) {
        
        // ========== VARIABILI NON UTILIZZATE ==========
        // Java 17 - variabile con nome
        try {
            int result = 10 / 0;
        } catch (ArithmeticException e) {
            // 'e' non viene usato
        }
        
        // Java 21 - unnamed variable
        try {
            int result = 10 / 0;
        } catch (ArithmeticException _) {
            // La variabile è anonima, chiara intenzione
            System.out.println("Divisione per zero");
        }
        
        // ========== LAMBDA EXPRESSIONS ==========
        List<String> list = Arrays.asList("a", "b", "c");
        
        // Java 17 - parametro con nome
        list.forEach(item -> System.out.println("Elemento"));
        
        // Java 21 - unnamed parameter
        list.forEach(_ -> System.out.println("Elemento"));
        
        // ========== TRY-WITH-RESOURCES ==========
        // Java 17
        try (FileReader reader = new FileReader("file.txt")) {
            // reader non usato
        } catch (IOException e) {
            // e non usato
        }
        
        // Java 21
        try (FileReader _ = new FileReader("file.txt")) {
            // Risorsa anonima
        } catch (IOException _) {
            // Eccezione anonima
        }
        
        // ========== PATTERN MATCHING ==========
        Object obj = "test";
        
        // Java 21 - unnamed in pattern matching
        if (obj instanceof String _) {
            System.out.println("È una stringa");
        }
        
        // Nei record patterns
        record Punto(int x, int y) {}
        Punto p = new Punto(5, 10);
        
        if (p instanceof Punto(int x, int _)) {
            System.out.println("x = " + x);
        }
    }
}
Output:
Divisione per zero
Elemento
È una stringa
x = 5

Scoped Values (Preview)

Java 17 → ThreadLocal (mutabile, costoso)
Java 21 → Scoped Values (immutabili, efficienti)
import java.util.concurrent.*;

// Java 21 - Scoped Values (Preview)
// Alternativa più efficiente a ThreadLocal

public class ScopedValuesDemo {
    
    // ========== THREAD LOCAL (Java 17) ==========
    private static final ThreadLocal<String> threadLocal = new ThreadLocal<>();
    
    static void threadLocalExample() {
        threadLocal.set("valore");
        String value = threadLocal.get();
        threadLocal.remove();  // Necessario per evitare memory leak
    }
    
    // ========== SCOPED VALUES (Java 21) ==========
    private static final ScopedValue<String> SCOPED = ScopedValue.newInstance();
    
    static void scopedValueExample() {
        // Bind immutabile
        ScopedValue.where(SCOPED, "valore").run(() -> {
            // Dentro questo scope, SCOPED.get() restituisce "valore"
            String value = SCOPED.get();
            System.out.println("Valore: " + value);
            
            // Operazioni annidate
            nestedOperation();
        });
        // Fuori dallo scope, SCOPED non è definito
    }
    
    static void nestedOperation() {
        // Eredita il valore dallo scope padre
        String value = SCOPED.get();
        System.out.println("Nested: " + value);
    }
    
    // ========== CONCORRENZA ==========
    static void concurrentExample() throws Exception {
        try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
            for (int i = 0; i < 10; i++) {
                final int id = i;
                executor.submit(() -> {
                    ScopedValue.where(SCOPED, "User-" + id).run(() -> {
                        processRequest();
                    });
                });
            }
        }
    }
    
    static void processRequest() {
        String userId = SCOPED.get();
        System.out.println("Processing request for: " + userId);
    }
    
    public static void main(String[] args) throws Exception {
        scopedValueExample();
        concurrentExample();
    }
}
Output:
Valore: valore
Nested: valore
Processing request for: User-0
Processing request for: User-1
...

API Novità

Nuove API introdotte in Java 21
import java.util.*;
import java.util.concurrent.*;
import java.time.*;

public class NuoveApiJava21 {
    public static void main(String[] args) {
        
        // ========== 1. SEQUENCED COLLECTIONS ==========
        SequencedCollection<String> list = new ArrayList<>(List.of("A", "B", "C"));
        System.out.println("First: " + list.getFirst());  // A
        System.out.println("Last: " + list.getLast());    // C
        
        // ========== 2. MATH API ==========
        // Divisione con arrotondamento verso zero
        long result1 = Math.divideExact(10, 3);   // 3
        // Clamping - limita un valore tra min e max
        int clamped = Math.clamp(15, 10, 20);     // 15
        int clamped2 = Math.clamp(5, 10, 20);     // 10
        int clamped3 = Math.clamp(25, 10, 20);    // 20
        
        // ========== 3. COLLECTION FACTORY METHODS ==========
        // Set.of, List.of, Map.of già esistenti, ora più efficienti
        
        // ========== 4. FUTURE TASKS ==========
        // Nuovi metodi in Future
        Future<String> future = CompletableFuture.completedFuture("done");
        // future.resultNow() - ottiene il risultato se completato
        // future.exceptionNow() - ottiene l'eccezione se fallito
        // future.state() - ottiene lo stato (RUNNING, SUCCESS, FAILED)
        
        // ========== 5. TIME API ==========
        LocalDate date = LocalDate.of(2024, 10, 15);
        DayOfWeek dow = DayOfWeek.from(date);
        // Nuovi metodi per formattazione
        String formatted = date.format(java.time.format.DateTimeFormatter.ISO_LOCAL_DATE);
        
        // ========== 6. ZONED DATE TIME ==========
        ZonedDateTime now = ZonedDateTime.now();
        // Metodi per conversione tra fusi orari
        ZonedDateTime utc = now.withZoneSameInstant(ZoneId.of("UTC"));
        
        System.out.println("Clamped: " + clamped);
        System.out.println("Formatted date: " + formatted);
    }
}
Output:
First: A
Last: C
Clamped: 15
Formatted date: 2024-10-15

// ==================== API RIMOSSE ====================

// 1. Finalizer (rimosso in Java 18)

// 4. URL public constructors (deprecati)
// new URL("http", "example.com", 80, "/path") - deprecato

// ==================== API DEPRECATE ====================

// 1. Locale.ISO_3_LANGUAGE e ISO_3_COUNTRY (deprecati)
// Usare Locale.forLanguageTag() invece

// 2. Thread.destroy() e Thread.stop(Throwable) - deprecati

// 3. Compiler.enable() e Compiler.disable() - deprecati

// ==================== DA AGGIORNARE ====================

// ❌ Vecchio codice (Java 17)
// Thread t = new Thread();
// t.stop();

// ✅ Nuovo codice (Java 21)
// Thread t = new Thread();
// t.interrupt();
// t.join();
Attenzione: Il Security Manager è stato deprecato per la rimozione. Il codice che lo utilizza deve essere migrato ad alternative moderne.
