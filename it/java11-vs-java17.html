Switch Expression

Java 11 → Solo switch statement
Java 17 → Switch expression + yield
Java 11
String giorno = "LUNEDI";
String tipo;
switch (giorno) {
    case "SABATO":
    case "DOMENICA":
        tipo = "Weekend";
        break;
    case "LUNEDI":
    case "MARTEDI":
    case "MERCOLEDI":
    case "GIOVEDI":
    case "VENERDI":
        tipo = "Lavorativo";
        break;
    default:
        tipo = "Sconosciuto";
}
System.out.println(tipo);
Java 17 NUOVO
String giorno = "LUNEDI";
String tipo = switch (giorno) {
    case "SABATO", "DOMENICA" -> "Weekend";
    case "LUNEDI", "MARTEDI", 
         "MERCOLEDI", "GIOVEDI", 
         "VENERDI" -> "Lavorativo";
    default -> "Sconosciuto";
};
System.out.println(tipo);

// Con yield per blocchi di codice
String risultato = switch (giorno) {
    case "LUNEDI" -> {
        System.out.println("Inizio settimana");
        yield "Lunedì";
    }
    default -> "Altro";
};
Java 17 vantaggio: 5 righe vs 16 righe per lo stesso risultato!

Record Class (Data Carrier)

Java 11 → Classi con boilerplate (~50 righe)
Java 17 → Record (1 riga)
Java 11
public class Persona {
    private final String nome;
    private final int eta;
    
    public Persona(String nome, int eta) {
        this.nome = nome;
        this.eta = eta;
    }
    
    public String getNome() { return nome; }
    public int getEta() { return eta; }
    
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Persona)) return false;
        Persona that = (Persona) o;
        return eta == that.eta && 
               Objects.equals(nome, that.nome);
    }
    
    @Override
    public int hashCode() {
        return Objects.hash(nome, eta);
    }
    
    @Override
    public String toString() {
        return "Persona{nome='" + nome + "', eta=" + eta + "}";
    }
}
Java 17 NUOVO
public record Persona(String nome, int eta) {}

// Uso
Persona p = new Persona("Mario", 30);
System.out.println(p.nome());  // getter automatico
System.out.println(p);  // toString() automatico

// Record con costruttore compatto (validazione)
public record Utente(String email, int eta) {
    public Utente {
        if (eta < 0) 
            throw new IllegalArgumentException("Età negativa");
        if (!email.contains("@")) 
            throw new IllegalArgumentException("Email invalida");
    }
}

// Record con metodi aggiuntivi
public record Punto(int x, int y) {
    public double distanzaDaOrigine() {
        return Math.sqrt(x * x + y * y);
    }
}
Java 17 vantaggio: 1 riga di codice vs ~50 righe per un DTO immutabile!

Pattern Matching per instanceof

Java 11 → Cast esplicito
Java 17 → Pattern binding diretto
Java 11
Object obj = "Ciao Mondo";
if (obj instanceof String) {
    String s = (String) obj;  // cast esplicito!
    System.out.println(s.toUpperCase());
}

// Con condizioni aggiuntive
if (obj instanceof String) {
    String s = (String) obj;
    if (s.length() > 5) {
        System.out.println("Lunga: " + s);
    }
}
Java 17 NUOVO
Object obj = "Ciao Mondo";
if (obj instanceof String s) {
    System.out.println(s.toUpperCase());
}

// Con condizioni aggiuntive (&&)
if (obj instanceof String s && s.length() > 5) {
    System.out.println("Lunga: " + s);
}

// Pattern matching in switch (Java 17 preview, 21 stabile)
Object valore = "Test";
String risultato = switch (valore) {
    case Integer i -> "Intero: " + i;
    case String s -> "Stringa: " + s;
    case null -> "Null";
    default -> "Sconosciuto";
};

Sealed Class (Classi sigillate)

Java 11 → Ereditarietà aperta (qualsiasi classe può estendere)
Java 17 → Controllo preciso delle sottoclassi permesse
// Java 17 - Sealed Class
public sealed class Forma 
    permits Cerchio, Rettangolo, Triangolo {
    public abstract double area();
}

// Le sottoclassi devono essere final, sealed o non-sealed
public final class Cerchio extends Forma {
    private final double raggio;
    
    public Cerchio(double raggio) { this.raggio = raggio; }
    
    @Override
    public double area() { 
        return Math.PI * raggio * raggio; 
    }
}

public sealed class Rettangolo extends Forma permits Quadrato {
    private final double base;
    private final double altezza;
    
    public Rettangolo(double base, double altezza) {
        this.base = base;
        this.altezza = altezza;
    }
    
    @Override
    public double area() { 
        return base * altezza; 
    }
}

public final class Quadrato extends Rettangolo {
    public Quadrato(double lato) {
        super(lato, lato);
    }
}

public non-sealed class Triangolo extends Forma {
    // non-sealed permette estensioni illimitate
    private final double base;
    private final double altezza;
    
    public Triangolo(double base, double altezza) {
        this.base = base;
        this.altezza = altezza;
    }
    
    @Override
    public double area() { 
        return (base * altezza) / 2; 
    }
}

// Tentare di estendere Forma con una classe non permessa causa errore!
// public class Pentagon extends Forma {}  // ERRORE di compilazione!
Java 17 vantaggio: Controllo esplicito su quali classi possono estendere

var in Lambda Parameters

Java 11 → Tipo implicito senza annotazioni
Java 17 → var con annotazioni nei lambda
import javax.validation.constraints.NotNull;
import java.util.*;

public class VarLambdaDemo {
    public static void main(String[] args) {
        List<String> nomi = Arrays.asList("Alice", "Bob", "Charlie");
        
        // ========== Java 11 ==========
        // Lambda con tipo inferito (senza var)
        nomi.sort((a, b) -> a.compareTo(b));
        
        // Non è possibile usare annotazioni nei parametri lambda
        // nomi.sort((@NotNull a, @NotNull b) -> a.compareTo(b));  // ERRORE!
        
        // ========== Java 17 ==========
        // var può essere usato nei parametri lambda
        nomi.sort((var a, var b) -> a.compareTo(b));
        
        // Con annotazioni (utile per validazione) - solo con var!
        nomi.sort((@NotNull var a, @NotNull var b) -> a.compareTo(b));
        
        // var permette di specificare il tipo quando serve
        Function<String, Integer> lunghezza = (var s) -> s.length();
        
        // Utile con tipi generici complessi
        BiFunction<List<String>, Integer, String> prendi = 
            (var lista, var index) -> lista.get(index);
    }
}

Nuovi metodi su String

Java 11 → strip(), isBlank(), lines(), repeat()
Java 17 → Stessi metodi (più maturi)
String testo = "  \t Hello World \n  ";

// strip() vs trim() - differenza importante!
String strip = testo.strip();   // "Hello World" (rimuove spazi Unicode)
String trim = testo.trim();      // "Hello World" (solo ASCII)

// isBlank() - verifica se stringa è vuota o solo spazi
"   ".isBlank();  // true
"".isBlank();      // true

// lines() - split per linee (più efficiente di split("\n"))
String multilinea = "riga1\nriga2\nriga3";
multilinea.lines().forEach(System.out::println);

// repeat() - ripete la stringa
"Java".repeat(3);  // "JavaJavaJava"

// formatted() - formattazione stile printf (Java 15+)
String nome = "Mario";
int eta = 30;
String msg = "Nome: %s, Età: %d".formatted(nome, eta);
System.out.println(msg);  // "Nome: Mario, Età: 30"

// indent() - aggiunge/rimuove indentazione (Java 12+)
String indentato = "testo".indent(4);  // "    testo\n"

// transform() - trasforma la stringa (Java 12+)
String risultato = "hello".transform(s -> s.toUpperCase());
System.out.println(risultato);  // "HELLO"

HTTP Client

Java 11 → HttpClient moderno (introdotto)
Java 17 → HttpClient migliorato (WebSocket, HTTP/2)
import java.net.http.*;
import java.net.URI;
import java.time.Duration;
import java.util.concurrent.*;

public class HttpClientDemo {
    public static void main(String[] args) throws Exception {
        
        // HttpClient moderno (introdotto in Java 11, migliorato in Java 17)
        HttpClient client = HttpClient.newBuilder()
            .version(HttpClient.Version.HTTP_2)
            .connectTimeout(Duration.ofSeconds(10))
            .followRedirects(HttpClient.Redirect.NORMAL)
            .build();
        
        // Richiesta GET sincrona
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create("https://api.example.com/users/1"))
            .timeout(Duration.ofSeconds(30))
            .header("Content-Type", "application/json")
            .GET()
            .build();
        
        HttpResponse<String> response = client.send(request, 
            HttpResponse.BodyHandlers.ofString());
        System.out.println("Status: " + response.statusCode());
        System.out.println("Body: " + response.body());
        
        // Richiesta asincrona (non bloccante)
        CompletableFuture<HttpResponse<String>> future = client.sendAsync(
            request, 
            HttpResponse.BodyHandlers.ofString()
        );
        future.thenAccept(resp -> System.out.println(resp.body()));
        
        // Richiesta POST con JSON
        String json = "{\"nome\":\"Mario\",\"eta\":30}";
        HttpRequest postRequest = HttpRequest.newBuilder()
            .uri(URI.create("https://api.example.com/users"))
            .header("Content-Type", "application/json")
            .POST(HttpRequest.BodyPublishers.ofString(json))
            .build();
        
        HttpResponse<String> postResponse = client.send(postRequest, 
            HttpResponse.BodyHandlers.ofString());
        
        // WebSocket support (Java 11+)
        WebSocket ws = client.newWebSocketBuilder()
            .buildAsync(URI.create("ws://example.com/socket"), new WebSocket.Listener() {})
            .join();
    }
}

Garbage Collector e Performance

Java 11 → G1GC (default), ZGC sperimentale
Java 17 → ZGC stabile, miglioramenti G1GC, Shenandoah
Java 11:
- G1GC di default
- ZGC richiede flag sperimentale: -XX:+UnlockExperimentalVMOptions -XX:+UseZGC
- Pause GC maggiori
Java 17:
- ZGC stabile: -XX:+UseZGC (nessun flag sperimentale)
- Shenandoah GC disponibile: -XX:+UseShenandoahGC
- Pause GC inferiori a 10ms anche per heap da terabyte
- Compressed oops supportato per ZGC
- Migliore integrazione con container
Confronto GC:
- G1GC: bilanciato, buono per la maggior parte dei casi
- ZGC: pause molto basse, ideale per grandi heap
- Shenandoah: pause basse, concorrente

API e Funzionalità Rimosse

Rimosse tra Java 11 e Java 17
// ==================== API RIMOSSE IN JAVA 17 ====================

// 1. Applet API (deprecata da Java 9, rimossa in Java 17)
// - java.applet.Applet
// - javax.swing.JApplet

// 2. RMI Activation (deprecato in Java 15, rimosso in Java 17)
// - java.rmi.activation.Activatable
// - java.rmi.activation.ActivationGroup

// 3. Security Manager (@Deprecated(forRemoval=true))
// System.setSecurityManager(...) - deprecato, verrà rimosso

// 4. ConcurrentHashMap - vecchi metodi
// ConcurrentHashMap.keySet().map(...) - alcune varianti deprecate

// ==================== API DEPRECATE PER RIMOZIONE ====================

// Finalizer (deprecato da Java 9)
@Override
@Deprecated(since="9", forRemoval=true)
protected void finalize() {
    // Non usare più! Usa Cleaner o try-with-resources
}

// Thread.stop(), Thread.suspend(), Thread.resume()
Thread t = new Thread();
t.stop();  // @Deprecated(forRemoval=true) - non usare!

// URL constructor (deprecato per rimozione)
new URL("http", "example.com", 80, "/path");  // Usare URI.toURL() invece

// System.runFinalizersOnExit()
System.runFinalizersOnExit(true);  // Deprecato, pericoloso!
Attenzione: Il Security Manager verrà rimosso completamente in una versione futura. Migrare il codice che lo utilizza.

Guida alla Migrazione da Java 11 a Java 17

FaseAzioni consigliateDettagli
1️⃣ Aggiorna la toolchain Maven/Gradle: aggiorna plugin compiler, surefire, failsafe
2️⃣ Verifica dipendenze Controlla che tutte le librerie supportino Java 17
3️⃣ Rimuovi codice deprecato Elimina chiamate a finalize(), stop(), suspend()
4️⃣ Testa con --illegal-access=deny Identifica accessi a API interne
5️⃣ Adotta nuove feature Records, Switch expression, Text blocks, Pattern matching
