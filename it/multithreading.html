Multithreading e Concorrenza in Java

Thread, Runnable, sincronizzazione, ExecutorService, Concurrent Collections e best practices

Cos'è il Multithreading?

Il multithreading è la capacità di un programma di eseguire più thread (sotto-processi) contemporaneamente. Java supporta nativamente il multithreading, permettendo di sfruttare al meglio i processori multi-core e di creare applicazioni più reattive.
Vantaggi del multithreading:
- Prestazioni: sfrutta processori multi-core
- Reattività: UI non si blocca durante operazioni lunghe
- Condivisione risorse: thread condividono memoria
- Scalabilità: più richieste gestite simultaneamente

Creazione di Thread

Tre modi principali per creare thread

📌 MODO 1: Estendere Thread

class ThreadEsteso extends Thread {
    private String nome;
    
    public ThreadEsteso(String nome) {
        this.nome = nome;
    }
    
    @Override
    public void run() {
        for (int i = 1; i <= 5; i++) {
            System.out.println(nome + " - step " + i);
            try {
                Thread.sleep(500);
            } catch (InterruptedException e) {
                System.out.println(nome + " interrotto");
            }
        }
    }
}

// Utilizzo
Thread t1 = new ThreadEsteso("Thread-1");
Thread t2 = new ThreadEsteso("Thread-2");
t1.start();
t2.start();

📌 MODO 2: Implementare Runnable (consigliato)

class RunnableImpl implements Runnable {
    private String nome;
    
    public RunnableImpl(String nome) {
        this.nome = nome;
    }
    
    @Override
    public void run() {
        for (int i = 1; i <= 5; i++) {
            System.out.println(nome + " [Runnable] - " + i);
            try {
                Thread.sleep(400);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
    }
}

// Utilizzo
Thread t3 = new Thread(new RunnableImpl("Runnable-1"));
t3.start();

📌 MODO 3: Lambda Expression (Java 8+)

Thread t4 = new Thread(() -> {
    for (int i = 1; i <= 3; i++) {
        System.out.println("Lambda: " + i);
    }
});
t4.start();
Nota: Implementare Runnable è preferibile perché Java non supporta ereditarietà multipla. Inoltre permette di usare lambda expressions.

Ciclo di Vita del Thread

NEW → RUNNABLE → RUNNING → BLOCKED/WAITING/TIMED_WAITING → TERMINATED
public class CicloVitaDemo {
    public static void main(String[] args) throws InterruptedException {
        Thread thread = new Thread(() -> {
            System.out.println("Thread in esecuzione...");
            try {
                Thread.sleep(1000);  // TIMED_WAITING
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println("Thread terminato");
        });
        
        System.out.println("Stato dopo creazione: " + thread.getState());  // NEW
        
        thread.start();
        System.out.println("Stato dopo start: " + thread.getState());  // RUNNABLE
        
        Thread.sleep(500);
        System.out.println("Stato durante sleep: " + thread.getState());  // TIMED_WAITING
        
        thread.join();
        System.out.println("Stato dopo join: " + thread.getState());  // TERMINATED
    }
}
Output:
Stato dopo creazione: NEW
Stato dopo start: RUNNABLE
Thread in esecuzione...
Stato durante sleep: TIMED_WAITING
Thread terminato
Stato dopo join: TERMINATED

Sincronizzazione e Race Condition

Attenzione: senza sincronizzazione, più thread possono corrompere i dati condivisi

📌 ESEMPIO SENZA SINCRONIZZAZIONE (RACE CONDITION)

class ContatoreNonSincronizzato {
    private int conteggio = 0;
    
    public void incrementa() {
        conteggio++;  // Non è atomico!
    }
    
    public int getConteggio() {
        return conteggio;
    }
}

// Problema: il risultato finale potrebbe essere minore di 10000!

📌 ESEMPIO CON SINCRONIZZAZIONE (CORRETTO)

class ContatoreSincronizzato {
    private int conteggio = 0;
    
    // Metodo sincronizzato (solo un thread alla volta)
    public synchronized void incrementa() {
        conteggio++;
    }
    
    public synchronized int getConteggio() {
        return conteggio;
    }
}

class Lavoratore extends Thread {
    private ContatoreSincronizzato contatore;
    
    public Lavoratore(ContatoreSincronizzato contatore) {
        this.contatore = contatore;
    }
    
    @Override
    public void run() {
        for (int i = 0; i < 1000; i++) {
            contatore.incrementa();
        }
    }
}

public class SincronizzazioneDemo {
    public static void main(String[] args) throws InterruptedException {
        ContatoreSincronizzato contatore = new ContatoreSincronizzato();
        Thread[] threads = new Thread[10];
        
        for (int i = 0; i < 10; i++) {
            threads[i] = new Lavoratore(contatore);
            threads[i].start();
        }
        
        for (Thread t : threads) {
            t.join();  // Attende la terminazione
        }
        
        System.out.println("Conteggio finale: " + contatore.getConteggio());
        // Dovrebbe essere 10 * 1000 = 10000
    }
}
Output: Conteggio finale: 10000
Alternative a synchronized:
- volatile per variabili semplici
- AtomicInteger, AtomicLong, ecc.
- ReentrantLock per maggiore flessibilità

ExecutorService e Thread Pool

Gestione professionale dei thread
import java.util.concurrent.*;
import java.util.List;
import java.util.ArrayList;

public class ExecutorServiceDemo {
    public static void main(String[] args) throws Exception {
        // 1. FixedThreadPool - pool con numero fisso di thread
        ExecutorService fixedPool = Executors.newFixedThreadPool(5);
        
        // 2. CachedThreadPool - crea thread quando necessario, riusa quelli esistenti
        ExecutorService cachedPool = Executors.newCachedThreadPool();
        
        // 3. SingleThreadExecutor - un singolo thread sequenziale
        ExecutorService singlePool = Executors.newSingleThreadExecutor();
        
        // 4. ScheduledThreadPool - per task programmati
        ScheduledExecutorService scheduler = Executors.newScheduledThreadPool(2);
        
        // ========== ESECUZIONE TASK ==========
        // Esecuzione di task senza ritorno
        fixedPool.submit(() -> {
            System.out.println("Task eseguito da: " + Thread.currentThread().getName());
        });
        
        // Esecuzione con risultato (Callable)
        Callable<Integer> taskCallable = () -> {
            Thread.sleep(1000);
            return 42;
        };
        
        Future<Integer> future = fixedPool.submit(taskCallable);
        System.out.println("Risultato: " + future.get());
        
        // ========== TASK PROGRAMMATI ==========
        // Esegue dopo 2 secondi
        scheduler.schedule(() -> System.out.println("Task dopo 2 secondi"), 2, TimeUnit.SECONDS);
        
        // Esegue periodicamente ogni 3 secondi (dopo 1 secondo di ritardo iniziale)
        scheduler.scheduleAtFixedRate(() -> System.out.println("Task periodico"), 1, 3, TimeUnit.SECONDS);
        
        // ========== INVOKE ALL ==========
        List<Callable<String>> tasks = Arrays.asList(
            () -> "Task 1 completato",
            () -> "Task 2 completato",
            () -> "Task 3 completato"
        );
        
        List<Future<String>> risultati = fixedPool.invokeAll(tasks);
        for (Future<String> res : risultati) {
            System.out.println(res.get());
        }
        
        // ========== SHUTDOWN ==========
        fixedPool.shutdown();
        try {
            if (!fixedPool.awaitTermination(60, TimeUnit.SECONDS)) {
                fixedPool.shutdownNow();
            }
        } catch (InterruptedException e) {
            fixedPool.shutdownNow();
        }
        
        // scheduler.shutdown();  // Ricordati di shutdown anche lo scheduler
    }
}
Tipi di ExecutorService:
- newFixedThreadPool(n): pool con numero fisso di thread
- newCachedThreadPool(): crea thread quando necessario, riusa quelli esistenti
- newSingleThreadExecutor(): un singolo thread sequenziale
- newScheduledThreadPool(n): per task programmati

Thread Safety con Concurrent Collections

Collections thread-safe per ambienti concorrenti
import java.util.concurrent.*;
import java.util.*;

public class ConcurrentCollectionsDemo {
    public static void main(String[] args) throws InterruptedException {
        // HashMap NON è thread-safe!
        // Map<String, Integer> map = new HashMap<>();  // PROBLEMA!
        
        // ========== SOLUZIONI THREAD-SAFE ==========
        
        // 1. ConcurrentHashMap
        Map<String, Integer> concurrentMap = new ConcurrentHashMap<>();
        
        // 2. CopyOnWriteArrayList (ottimo per letture frequenti, scritture rare)
        List<Integer> copyOnWriteList = new CopyOnWriteArrayList<>();
        
        // 3. ConcurrentLinkedQueue (coda non bloccante)
        Queue<String> concurrentQueue = new ConcurrentLinkedQueue<>();
        
        // 4. BlockingQueue (coda bloccante)
        BlockingQueue<Integer> blockingQueue = new ArrayBlockingQueue<>(10);
        
        // ========== ESEMPIO PRODUCER-CONSUMER ==========
        BlockingQueue<Integer> coda = new LinkedBlockingQueue<>();
        
        // Producer
        Thread producer = new Thread(() -> {
            try {
                for (int i = 1; i <= 10; i++) {
                    coda.put(i);
                    System.out.println("📦 Prodotto: " + i);
                    Thread.sleep(100);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });
        
        // Consumer
        Thread consumer = new Thread(() -> {
            try {
                while (true) {
                    Integer val = coda.take();
                    System.out.println("📦 Consumato: " + val);
                    if (val == 10) break;
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });
        
        producer.start();
        consumer.start();
        producer.join();
        consumer.join();
        
        // ========== ESEMPIO CONCURRENT HASHMAP ==========
        System.out.println("\n=== ConcurrentHashMap esempio ===");
        ConcurrentHashMap<String, Integer> voti = new ConcurrentHashMap<>();
        
        // Operazioni atomiche
        voti.put("Alice", 85);
        voti.put("Bob", 70);
        
        // putIfAbsent - inserisce solo se la chiave non esiste
        voti.putIfAbsent("Alice", 100);  // Non fa nulla
        voti.putIfAbsent("Diana", 90);   // Inserisce
        
        // replace condizionale
        voti.replace("Bob", 70, 75);     // Sostituisce solo se il valore corrente è 70
        
        // compute
        voti.compute("Alice", (k, v) -> v + 5);
        
        System.out.println("Voti: " + voti);
    }
}

wait(), notify(), notifyAll()

Comunicazione tra thread
class MessageQueue {
    private String message;
    private boolean empty = true;
    
    public synchronized String take() throws InterruptedException {
        while (empty) {
            wait();  // Attende che il messaggio sia disponibile
        }
        empty = true;
        notifyAll();  // Notifica i producer
        return message;
    }
    
    public synchronized void put(String message) throws InterruptedException {
        while (!empty) {
            wait();  // Attende che il messaggio venga consumato
        }
        empty = false;
        this.message = message;
        notifyAll();  // Notifica i consumer
    }
}

public class WaitNotifyDemo {
    public static void main(String[] args) {
        MessageQueue queue = new MessageQueue();
        
        // Consumer
        Thread consumer = new Thread(() -> {
            try {
                for (int i = 0; i < 5; i++) {
                    String msg = queue.take();
                    System.out.println("📩 Ricevuto: " + msg);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });
        
        // Producer
        Thread producer = new Thread(() -> {
            String[] messaggi = {"Msg1", "Msg2", "Msg3", "Msg4", "Msg5"};
            try {
                for (String msg : messaggi) {
                    queue.put(msg);
                    System.out.println("📤 Inviato: " + msg);
                    Thread.sleep(500);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });
        
        consumer.start();
        producer.start();
    }
}
Output:
📤 Inviato: Msg1
📩 Ricevuto: Msg1
📤 Inviato: Msg2
📩 Ricevuto: Msg2
...
Importante: wait(), notify() e notifyAll() devono essere chiamati solo all'interno di blocchi sincronizzati (synchronized)!

CompletableFuture (Programmazione Asincrona)

Java 8+ - Pipeline asincrone non bloccanti
import java.util.concurrent.*;
import java.util.List;

public class CompletableFutureDemo {
    public static void main(String[] args) throws Exception {
        // ========== ESECUZIONE ASINCRONA SEMPLICE ==========
        CompletableFuture<String> future = CompletableFuture.supplyAsync(() -> {
            try { Thread.sleep(1000); } catch (InterruptedException e) {}
            return "Risultato";
        });
        
        future.thenAccept(result -> System.out.println("Ricevuto: " + result));
        
        // ========== PIPELINE DI OPERAZIONI ==========
        CompletableFuture.supplyAsync(() -> "Hello")
            .thenApply(s -> s + " World")
            .thenApply(String::toUpperCase)
            .thenAccept(System.out::println);
        
        // ========== COMBINAZIONE DI PIÙ FUTURES ==========
        CompletableFuture<Integer> future1 = CompletableFuture.supplyAsync(() -> 10);
        CompletableFuture<Integer> future2 = CompletableFuture.supplyAsync(() -> 20);
        
        CompletableFuture<Integer> somma = future1.thenCombine(future2, (a, b) -> a + b);
        System.out.println("Somma: " + somma.get());
        
        // ========== GESTIONE ERRORI ==========
        CompletableFuture.supplyAsync(() -> {
            if (true) throw new RuntimeException("Errore!");
            return "OK";
        }).exceptionally(ex -> {
            System.out.println("Errore: " + ex.getMessage());
            return "Fallback";
        }).thenAccept(System.out::println);
        
        // ========== ATTESA DI TUTTI I FUTURES ==========
        List<CompletableFuture<String>> futures = List.of(
            CompletableFuture.supplyAsync(() -> "Task 1"),
            CompletableFuture.supplyAsync(() -> "Task 2"),
            CompletableFuture.supplyAsync(() -> "Task 3")
        );
        
        CompletableFuture.allOf(futures.toArray(new CompletableFuture[0]))
            .thenRun(() -> System.out.println("Tutti i task completati"));
        
        // ========== CORSA TRA FUTURES ==========
        CompletableFuture<String> veloce = CompletableFuture.supplyAsync(() -> {
            sleep(100); return "Veloce";
        });
        CompletableFuture<String> lento = CompletableFuture.supplyAsync(() -> {
            sleep(500); return "Lento";
        });
        
        CompletableFuture<Object> primo = CompletableFuture.anyOf(veloce, lento);
        System.out.println("Primo completato: " + primo.get());
    }
    
    private static void sleep(int ms) {
        try { Thread.sleep(ms); } catch (InterruptedException e) {}
    }
}
Output:

Somma: 30
Errore: java.lang.RuntimeException: Errore!
Fallback
Ricevuto: Risultato
Tutti i task completati
Primo completato: Veloce

Lock e ReentrantLock

Alternativa più flessibile a synchronized
import java.util.concurrent.locks.*;
import java.util.*;

public class LockDemo {
    
    // ========== REENTRANT LOCK ==========
    static class ContatoreConLock {
        private int conteggio = 0;
        private final Lock lock = new ReentrantLock();
        
        public void incrementa() {
            lock.lock();
            try {
                conteggio++;
            } finally {
                lock.unlock();  // Importante: sempre in finally!
            }
        }
        
        public int getConteggio() {
            lock.lock();
            try {
                return conteggio;
            } finally {
                lock.unlock();
            }
        }
        
        // TryLock - non bloccante
        public boolean tryIncrementa() {
            if (lock.tryLock()) {
                try {
                    conteggio++;
                    return true;
                } finally {
                    lock.unlock();
                }
            }
            return false;
        }
        
        // TryLock con timeout
        public boolean tryIncrementaConTimeout(long timeout, TimeUnit unit) throws InterruptedException {
            if (lock.tryLock(timeout, unit)) {
                try {
                    conteggio++;
                    return true;
                } finally {
                    lock.unlock();
                }
            }
            return false;
        }
    }
    
    // ========== READ WRITE LOCK ==========
    static class Cache {
        private final Map<String, String> data = new HashMap<>();
        private final ReadWriteLock readWriteLock = new ReentrantReadWriteLock();
        private final Lock readLock = readWriteLock.readLock();
        private final Lock writeLock = readWriteLock.writeLock();
        
        public String get(String key) {
            readLock.lock();
            try {
                return data.get(key);
            } finally {
                readLock.unlock();
            }
        }
        
        public void put(String key, String value) {
            writeLock.lock();
            try {
                data.put(key, value);
                System.out.println("Inserito: " + key + "=" + value);
            } finally {
                writeLock.unlock();
            }
        }
        
        public void clear() {
            writeLock.lock();
            try {
                data.clear();
                System.out.println("Cache svuotata");
            } finally {
                writeLock.unlock();
            }
        }
    }
    
    public static void main(String[] args) throws InterruptedException {
        // Test ReentrantLock
        ContatoreConLock contatore = new ContatoreConLock();
        
        Thread[] threads = new Thread[10];
        for (int i = 0; i < 10; i++) {
            threads[i] = new Thread(() -> {
                for (int j = 0; j < 1000; j++) {
                    contatore.incrementa();
                }
            });
            threads[i].start();
        }
        
        for (Thread t : threads) {
            t.join();
        }
        System.out.println("Contatore con Lock: " + contatore.getConteggio());
        
        // Test ReadWriteLock
        Cache cache = new Cache();
        
        // Scrittori
        Thread writer = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                cache.put("key" + i, "value" + i);
            }
        });
        
        // Lettori (possono essere concorrenti)
        Thread reader1 = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                System.out.println("Reader1: " + cache.get("key" + i));
            }
        });
        
        Thread reader2 = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                System.out.println("Reader2: " + cache.get("key" + i));
            }
        });
        
        writer.start();
        writer.join();
        reader1.start();
        reader2.start();
    }
}
Output:
Contatore con Lock: 10000
Inserito: key0=value0
...
Reader1: value0
Reader2: value0
...
Vantaggi di Lock rispetto a synchronized:
- tryLock() non bloccante
- lockInterruptibly() interrompibile
- ReadWriteLock per ottimizzare letture concorrenti
- Più controllo sul timeout

Classi Atomiche (java.util.concurrent.atomic)

Operazioni atomiche senza sincronizzazione esplicita
import java.util.concurrent.atomic.*;
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class AtomicDemo {
    public static void main(String[] args) throws InterruptedException {
        // ========== ATOMIC INTEGER ==========
        AtomicInteger atomicInt = new AtomicInteger(0);
        
        System.out.println("=== AtomicInteger ===");
        System.out.println("Valore iniziale: " + atomicInt.get());
        
        // Operazioni atomiche
        atomicInt.incrementAndGet();      // 1
        atomicInt.addAndGet(5);           // 6
        atomicInt.compareAndSet(6, 10);   // CAS (Compare And Swap)
        
        System.out.println("Dopo operazioni: " + atomicInt.get());
        
        // getAndUpdate
        atomicInt.getAndUpdate(n -> n * 2);
        System.out.println("Dopo update: " + atomicInt.get());
        
        // ========== ATOMIC BOOLEAN ==========
        AtomicBoolean flag = new AtomicBoolean(false);
        boolean wasFalse = flag.compareAndSet(false, true);
        System.out.println("\n=== AtomicBoolean ===");
        System.out.println("CompareAndSet riuscito? " + wasFalse);
        System.out.println("Valore flag: " + flag.get());
        
        // ========== ATOMIC REFERENCE ==========
        AtomicReference<String> ref = new AtomicReference<>("init");
        ref.set("nuovo valore");
        ref.compareAndSet("nuovo valore", "aggiornato");
        System.out.println("\n=== AtomicReference ===");
        System.out.println("Valore reference: " + ref.get());
        
        // ========== ESEMPIO PRATICO: CONTATORE SENZA LOCK ==========
        System.out.println("\n=== Contatore con AtomicLong ===");
        AtomicLong contatore = new AtomicLong(0);
        
        ExecutorService executor = Executors.newFixedThreadPool(10);
        for (int i = 0; i < 1000; i++) {
            executor.submit(contatore::incrementAndGet);
        }
        
        executor.shutdown();
        Thread.sleep(1000);
        System.out.println("Contatore atomico: " + contatore.get());  // 1000
        
        // ========== LONG ADDER (più performante per alte concorrenze) ==========
        System.out.println("\n=== LongAdder (alte concorrenze) ===");
        LongAdder adder = new LongAdder();
        
        // 1000 thread che incrementano
        ExecutorService executor2 = Executors.newFixedThreadPool(10);
        for (int i = 0; i < 1000; i++) {
            executor2.submit(() -> adder.increment());
        }
        
        executor2.shutdown();
        Thread.sleep(1000);
        System.out.println("LongAdder sum: " + adder.sum());
        System.out.println("LongAdder longValue: " + adder.longValue());
        
        // ========== ATOMIC INTEGER ARRAY ==========
        System.out.println("\n=== AtomicIntegerArray ===");
        AtomicIntegerArray array = new AtomicIntegerArray(5);
        array.set(0, 10);
        array.set(1, 20);
        array.incrementAndGet(0);
        array.addAndGet(1, 5);
        System.out.println("Array[0]: " + array.get(0));
        System.out.println("Array[1]: " + array.get(1));
    }
}

Confronto prestazioni:
- AtomicInteger/Long: ottimo per pochi thread
- LongAdder: migliore per alte concorrenze (molti thread)
- compareAndSet() (CAS) è l'operazione fondamentale alla base di tutte le classi atomiche

Best Practices per il Multithreading

Regole fondamentali per scrivere codice concorrente sicuro

// ✅ Buona pratica
new Thread(() -> {
    // task
}).start();

// ==================== 2. USA EXECUTORService INVECE DI THREAD MANUALI ====================
ExecutorService executor = Executors.newFixedThreadPool(10);

// ==================== 3. RENDI IMMUTABILI GLI OGGETTI CONDIVISI ====================
public final class ImmutablePerson {
    private final String name;
    private final int age;
    
    public ImmutablePerson(String name, int age) {
        this.name = name;
        this.age = age;
    }
    public String getName() { return name; }
    public int getAge() { return age; }
}

// ==================== 5. MINIMIZZA LA SINCRONIZZAZIONE ====================
// Sincronizza solo le sezioni critiche, non interi metodi
public void increment() {

    }
}

private volatile boolean running = true;

    }
}

// ==================== 10. USA COMPLETABLE FUTURE PER OPERAZIONI ASINCRONE ====================
CompletableFuture.supplyAsync(() -> "Hello")
    .thenApply(String::toUpperCase)
    .thenAccept(System.out::println);
Attenzione ai Deadlock: Assicurati che i thread acquisiscano i lock nello stesso ordine per evitare deadlock!
