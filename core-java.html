Core Java

I fondamenti del linguaggio Java: dalla JVM alla sintassi base, tipi di dati, operatori e strutture di controllo

Cos'è Java?

Java è un linguaggio di programmazione object-oriented, platform-independent (grazie alla JVM) e fortemente tipizzato. Creato da Sun Microsystems nel 1995, è oggi uno dei linguaggi più utilizzati al mondo per applicazioni enterprise, mobile (Android) e web.
Caratteristiche principali di Java:
- Platform Independent: "Write Once, Run Anywhere" (WORA)
- Object-Oriented: tutto è organizzato in classi e oggetti
- Robust: gestione automatica della memoria (garbage collection)
- Secure: nessun puntatore esplicito, sandbox security
- Multithreaded: supporto nativo per programmazione concorrente

JVM, JRE, JDK

Java Virtual Machine, Java Runtime Environment, Java Development Kit

📌 JVM (Java Virtual Machine)

La JVM è il motore che esegue il bytecode Java. Fornisce l'ambiente di esecuzione indipendente dalla piattaforma.

📌 JRE (Java Runtime Environment)

Il JRE include la JVM e le librerie core necessarie per eseguire applicazioni Java (ma non per svilupparle).

📌 JDK (Java Development Kit)

Il JDK include il JRE più gli strumenti di sviluppo come compilatore (javac), debugger, ecc.

Operatori

Aritmetici, relazionali, logici, bitwise, di assegnamento

📌 Operatori Aritmetici

int a = 10, b = 3;
System.out.println("Somma: " + (a + b));       // 13
System.out.println("Differenza: " + (a - b));  // 7
System.out.println("Prodotto: " + (a * b));    // 30
System.out.println("Divisione: " + (a / b));   // 3 (divisione intera)
System.out.println("Modulo: " + (a % b));      // 1 (resto)

// Incremento/Decremento
int x = 5;
System.out.println(x++);  // 5 (post-incremento)
System.out.println(++x);  // 7 (pre-incremento)

📌 Operatori Relazionali

int a = 10, b = 20;
System.out.println(a == b);  // false (uguale)
System.out.println(a != b);  // true  (diverso)
System.out.println(a > b);   // false (maggiore)
System.out.println(a < b);   // true  (minore)
System.out.println(a >= b);  // false (maggiore o uguale)
System.out.println(a <= b);  // true  (minore o uguale)

📌 Operatori Logici

boolean p = true, q = false;
System.out.println(p && q);  // false (AND logico)
System.out.println(p || q);  // true  (OR logico)
System.out.println(!p);      // false (NOT logico)

// Short-circuit evaluation
boolean risultato = (a > 0) && (b / a > 10);  // b/a non valutato se a<=0

📌 Operatore Ternario

int max = (a > b) ? a : b;  // se a>b restituisce a, altrimenti b

📌 Cicli

// for loop
for (int i = 0; i < 5; i++) {
    System.out.print(i + " ");
}
// Output: 0 1 2 3 4

// while loop
int i = 0;
while (i < 5) {
    System.out.print(i + " ");
    i++;
}

// do-while (esegue almeno una volta)
int j = 0;
do {
    System.out.print(j + " ");
    j++;
} while (j < 5);

// enhanced for (for-each)
int[] numeri = {1, 2, 3, 4, 5};
for (int n : numeri) {
    System.out.print(n + " ");
}

// break e continue
for (int k = 0; k < 10; k++) {
    if (k == 5) break;      // esce dal ciclo
    if (k % 2 == 0) continue; // salta i pari
    System.out.print(k + " ");
}

Metodi

Dichiarazione, parametri, return, overload, varargs
public class MetodiDemo {
    
    // ========== METODO SENZA PARAMETRI E SENZA RETURN ==========
    public static void saluta() {
        System.out.println("Ciao da Java!");
    }
    
    // ========== METODO CON PARAMETRI E RETURN ==========
    public static int somma(int a, int b) {
        return a + b;
    }
    
    // ========== OVERLOAD (stesso nome, parametri diversi) ==========
    public static double somma(double a, double b) {
        return a + b;
    }
    
    public static int somma(int a, int b, int c) {
        return a + b + c;
    }
    
    // ========== VARARGS (numero variabile di argomenti) ==========
    public static int sommaVarargs(int... numeri) {
        int totale = 0;
        for (int n : numeri) {
            totale += n;
        }
        return totale;
    }
    
    // ========== METODO RICORSIVO ==========
    public static long fattoriale(int n) {
        if (n <= 1) return 1;
        return n * fattoriale(n - 1);
    }
    
    // ========== PASSAGGIO PARAMETRI ==========
    // Java è pass-by-value (le modifiche ai parametri non influenzano l'originale)
    public static void modificaPrimitivo(int x) {
        x = 100;  // non modifica la variabile originale
    }
    
    public static void modificaOggetto(StringBuilder sb) {
        sb.append(" aggiunto");  // modifica l'oggetto originale
    }
    
    public static void main(String[] args) {
        saluta();
        
        System.out.println("Somma 5+3: " + somma(5, 3));
        System.out.println("Somma 5.5+3.2: " + somma(5.5, 3.2));
        System.out.println("Somma 5+3+2: " + somma(5, 3, 2));
        System.out.println("Somma varargs: " + sommaVarargs(1, 2, 3, 4, 5));
        System.out.println("Fattoriale di 5: " + fattoriale(5));
        
        // Test passaggio parametri
        int valore = 10;
        modificaPrimitivo(valore);
        System.out.println("Valore dopo metodo: " + valore);  // 10 (invariato)
        
        StringBuilder sb = new StringBuilder("Test");
        modificaOggetto(sb);
        System.out.println("StringBuilder dopo metodo: " + sb);  // "Test aggiunto"
    }
}
Output:

Somma 5.5+3.2: 8.7
Somma 5+3+2: 10
Somma varargs: 15
Fattoriale di 5: 120

StringBuilder dopo metodo: Test aggiunto

Input/Output da Console

Scanner, System.out, System.err
import java.util.Scanner;

public class InputOutputDemo {
    public static void main(String[] args) {
        // ========== OUTPUT ==========
        System.out.println("Stampa con println (aggiunge newline)");
        System.out.print("Stampa con print (senza newline)");
        System.out.printf("Stampa formattata: %d, %s, %.2f%n", 42, "test", 3.14159);
        System.err.println("Questo è un messaggio di errore");
        
        // ========== INPUT CON SCANNER ==========
        Scanner scanner = new Scanner(System.in);
        
        System.out.println("\n--- Inserisci i tuoi dati ---");
        
        System.out.print("Inserisci il tuo nome: ");
        String nome = scanner.nextLine();
        
        System.out.print("Inserisci la tua età: ");
        int eta = scanner.nextInt();
        
        scanner.nextLine();  // consuma il newline rimasto
        
        System.out.print("Inserisci la tua città: ");
        String citta = scanner.nextLine();
        
        System.out.print("Inserisci il tuo stipendio: ");
        double stipendio = scanner.nextDouble();
        
        System.out.println("\n--- Riepilogo ---");
        System.out.printf("Nome: %s%n", nome);
        System.out.printf("Età: %d anni%n", eta);
        System.out.printf("Città: %s%n", citta);
        System.out.printf("Stipendio: €%.2f%n", stipendio);
        
        scanner.close();
    }
}
Esempio di interazione:

Nota: Ricorda sempre di chiudere lo Scanner con .close() per liberare le risorse!

Best Practices per Core Java

Regole fondamentali per scrivere codice Java di qualità
// ==================== 1. CONVENZIONI DI NOMENCLATURA ====================
// Classi: PascalCase (Iniziale maiuscola)
public class NomeClasse { }

// Metodi e variabili: camelCase (iniziale minuscola)
public void nomeMetodo() { }
int nomeVariabile = 10;

// Costanti: UPPER_SNAKE_CASE
public static final int MAX_VALUE = 100;

// Pacchetti: lowercase (tutto minuscolo)
package com.esempio.progetto;

// ==================== 2. COMMENTI E DOCUMENTAZIONE ====================
/**
 * Calcola la somma di due numeri interi.
 * 
 * @param a il primo numero
 * @param b il secondo numero
 * @return la somma di a e b
 * @throws IllegalArgumentException se i numeri sono negativi
 */
public int somma(int a, int b) {
    if (a < 0 || b < 0) {
        throw new IllegalArgumentException("Numeri non negativi");
    }
    return a + b;
}

// Commento su singola riga per spiegazioni brevi

/*
 * Commento su più righe
 * per spiegazioni più lunghe
 */

// ==================== 3. GESTIONE DEGLI ERRORI ====================
// ❌ Non ignorare le eccezioni
try {
    // operazione
} catch (Exception e) {
    // NON fare nulla - pessima pratica!
}

// ✅ Almeno logga l'errore
try {
    // operazione
} catch (Exception e) {
    System.err.println("Errore: " + e.getMessage());
    e.printStackTrace();
}

// ==================== 4. USO DI try-with-resources ====================
// ✅ Per risorse AutoCloseable
try (BufferedReader br = new BufferedReader(new FileReader("file.txt"))) {
    String line = br.readLine();
} catch (IOException e) {
    e.printStackTrace();
}
// br viene chiuso automaticamente

// ==================== 5. COMPARAZIONE STRINGHE ====================
// ❌ Non usare ==
if (str1 == str2) { }  // confronta riferimenti, non contenuto

// ✅ Usare equals
if (str1.equals(str2)) { }
if (str1.equalsIgnoreCase(str2)) { }  // case-insensitive

// ==================== 6. DICHIARAZIONE VARIABILI ====================
// ✅ Dichiarare le variabili il più vicino possibile al loro utilizzo
int risultato = 0;
for (int i = 0; i < 10; i++) {
    risultato += i;
}

// ==================== 7. EVITARE MAGIC NUMBERS ====================
// ❌ Magic number
if (status == 404) { }

// ✅ Costante con nome significativo
public static final int HTTP_NOT_FOUND = 404;
if (status == HTTP_NOT_FOUND) { }

// ==================== 8. STRINGBUILDER PER CONCATENAZIONI RIPETUTE ====================
// ❌ Inefficiente in loop
String result = "";
for (int i = 0; i < 1000; i++) {
    result += i;  // crea nuovi oggetti String
}

// ✅ Efficiente
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 1000; i++) {
    sb.append(i);
}
String result = sb.toString();

✓ Evita magic numbers (usa costanti)
✓ Usa StringBuilder per concatenazioni in loop
✓ Rendi le variabili final quando possibile (immutabilità)
✓ Usa nomi significativi per variabili e metodi
